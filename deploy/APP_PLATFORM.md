# Деплой на Timeweb App Platform (рекомендуемый способ)

Автодеплой по `git push`, бесплатный домен + HTTPS, откат в один клик. Для
не-разработчика — самый простой путь. Ручной сервер описан отдельно в
[deploy/README.md](./README.md) (запасной вариант).

## Что уже готово в репозитории

- Корневой **`docker-compose.yml`** — это файл для App Platform: три сервиса
  `proxy` + `web` + `api`. Платформа читает именно его из корня.
- `proxy` (Caddy) — первый сервис, его платформа публикует на домене с HTTPS;
  внутри он отправляет `/api/*` в бэкенд, остальное — на сайт. Поэтому сайт и
  API живут на **одном домене** — без проблем с CORS и cookies.
- `api` сам применяет миграции БД при каждом старте.
- БД (Timeweb PostgreSQL) и S3 — внешние, подключаются переменными.
- Redis на старте не используется (приложение работает и без него).

> Локальная разработка теперь использует `docker-compose.dev.yml`:
> `docker compose -f docker-compose.dev.yml up -d`.

## Шаг 1. Дать Timeweb доступ к репозиторию

В мастере «Добавить App»: **1. Тип → Docker → Docker Compose**. В разделе
**2. Репозиторий** выбери аккаунт `mojosay-del` и репозиторий
**`mojosay-del/ecoplatform`** (НЕ mircalculator). Если его нет в списке —
«Настроить на Github» и выдать доступ к ecoplatform. Ветка — **main**,
«Сборка по последнему коммиту» — включена (это и есть автодеплой).

## Шаг 2. Регион и мощность

- **Регион:** Санкт-Петербург (ближе к твоей БД и S3).
- **Конфигурация:** возьми **4 ГБ RAM** (1500 ₽/мес) или больше. ⚠️ На 2 ГБ
  сборка двух приложений (Next.js + NestJS) может упасть по памяти. После
  успешного запуска при желании уменьшишь.

## Шаг 3. Сеть — подключить БД приватно

В разделе **5. Сеть** добавь приложение в **приватную сеть**. К этой же сети
подключи кластер PostgreSQL `Fair Quail` (в настройках БД). Тогда приложение
ходит в базу по **приватному адресу** — базу не нужно открывать в интернет.
Приватный адрес БД подставишь в `DATABASE_URL` (шаг 4).

*Если с приватной сетью не получится — как запасной вариант оставь публичный
адрес БД `185.185.143.220`, но в разделе «Сеть» кластера разреши подключение
только этому приложению.*

## Шаг 4. Переменные (раздел 6 мастера → «Добавить»)

Добавь по одной. Значения — из вкладки «Подключение» БД и раздела S3.

| Переменная | Значение |
|---|---|
| `NEXT_PUBLIC_API_URL` | `https://ВАШ-ДОМЕН/api` (зашивается в сайт при сборке) |
| `WEB_ORIGIN` | `https://ВАШ-ДОМЕН` |
| `DATABASE_URL` | `postgresql://USER:PASS@ПРИВАТНЫЙ_ХОСТ:PORT/DB?schema=public&sslmode=require&connection_limit=20` |
| `JWT_ACCESS_SECRET` | `openssl rand -hex 32` (≥32 символов) |
| `JWT_REFRESH_SECRET` | другой `openssl rand -hex 32` |
| `S3_ENDPOINT` | `https://s3.twcstorage.ru` |
| `S3_REGION` | `ru-1` |
| `S3_BUCKET` | имя бакета (`bfe9c154-...`) |
| `S3_ACCESS_KEY_ID` / `S3_SECRET_ACCESS_KEY` | ключи S3 |
| `S3_PUBLIC_BASE_URL` | `https://s3.twcstorage.ru` |
| `PLATFORM_OWNER_EMAIL` | `mojosay@icloud.com` |
| `METRICS_BASIC_USER` / `METRICS_BASIC_PASSWORD` | любой логин + длинный пароль |
| `LOG_LEVEL` | `info` |
| `PWNED_PASSWORDS_CHECK_ENABLED` | `1` |
| `SEED_ADMIN_PASSWORD` / `SEED_DEMO_PASSWORD` | надёжные (≥12); временные, удалятся на шаге 8 |

> ⚠️ `NEXT_PUBLIC_API_URL` нужно знать на момент сборки. Поэтому сначала
> определись с доменом (шаг 6). Если сменишь домен позже — запусти пересборку
> приложения, чтобы новый адрес «зашился» в сайт.

## Шаг 5. Имя и проект, заказ

Раздел 7: имя любое, проект — **Экоплатформа**. Жми «Заказать». Платформа
соберёт образы и запустит контейнеры (первая сборка — несколько минут, следи за
логами сборки).

## Шаг 6. Домен и проверка

1. Сначала приложение получит технический домен с HTTPS — на нём можно
   проверить, что всё поднялось.
2. Привяжи свой домен **ecoplatform.pro** к приложению (раздел домена в панели);
   SSL выпустится автоматически.
3. Проверь здоровье API: открой `https://ВАШ-ДОМЕН/api/ready` — должно быть «ok».

## Шаг 7. Юр-документы (seed) — один раз

Открой в панели приложения **консоль** сервиса `api` и выполни:

```bash
pnpm seed
```

Создаст 5 обязательных юр-документов (без них регистрация/согласия не работают).

## Шаг 8. Стать первым админом

1. Зайди на сайт, **зарегистрируйся** своей почтой и паролём (рекомендуется
   `mojosay@icloud.com`).
2. В консоли сервиса `api` сначала «вхолостую» (ничего не меняет):
   ```bash
   pnpm exec ts-node prisma/scripts/promote-first-admin.ts
   ```
   Если отчёт верный — примени:
   ```bash
   PROMOTE_FIRST_ADMIN_WRITE=1 pnpm exec ts-node prisma/scripts/promote-first-admin.ts
   ```
   ⚠️ Запускать только сейчас, на чистой базе (скрипт оставляет лишь владельца).

## Как выкатывать обновления потом

Просто `git push` в `main` с ноутбука — App Platform сам пересоберёт и
перевыкатит приложение. Миграции БД применятся автоматически. Откат на прошлую
версию — кнопкой в панели.

## Если сборка/запуск упали — смотрим логи

В панели приложения есть логи сборки и логи каждого сервиса. Частое:
- **OOM при сборке** → увеличить RAM (шаг 2).
- **API не стартует** → короткие `JWT_*` или неверный `DATABASE_URL`.
- **Пустые согласия / нет входа** → не выполнен `pnpm seed` (шаг 7).
