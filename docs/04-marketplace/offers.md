## 11. Ценовое предложение: общая логика

### 11.1. Определение
Ценовое предложение (далее — предложение) — это закрытое коммерческое предложение покупателя по активному объявлению продавца с указанием цены и условий.

### 11.2. Закрытость предложений
Предложения закрытые: каждый покупатель видит только собственное предложение по конкретному объявлению. Покупатель не видит:
- количества других поступивших предложений
- диапазона цен по другим предложениям
- собственного места в списке относительно других предложений

Продавец видит все поступившие предложения одновременно, в одном списке, с сортировкой по цене (наиболее выгодное сверху).

### 11.3. Скрытие наименования компании до принятия
До момента принятия предложения продавцом наименование компании-покупателя не отображается в карточке предложения, и наименование компании-продавца не отображается в карточке отправленного предложения у покупателя. В поле наименования организации отображается текст «Наименование пока скрыто» с пояснением «Раскроем после принятия предложения».

Прочие атрибуты (рейтинг, количество отзывов, тип компании) отображаются в обычном режиме.

### 11.4. Ограничение на комментарии в предложении
Ценовое предложение не содержит поля для произвольного текстового комментария покупателя. Все условия, не укладывающиеся в формализованные поля предложения, обсуждаются между продавцом и покупателем по телефону после принятия предложения.

Запрет комментариев обусловлен риском обхода скрытия контактных данных.

### 11.5. Право делать предложения
Предложение по объявлению может сделать любая зарегистрированная компания типа «трейдер» или «переработчик» в рамках своего кабинета и при условии активной подписки.

Одна компания может сделать только одно активное предложение по одному объявлению. Для изменения условий используется редактирование существующего предложения в соответствии с разделом 13.

## 12. Ценовое предложение: состав полей

### 12.1. Обязательные поля
Каждое ценовое предложение обязательно содержит:
- объявление, к которому относится предложение
- покупатель (компания, подставляется автоматически)
- ответственный сотрудник внутри компании-покупателя
- условие цены — одно из двух значений:
  - «цена с места» (покупатель берёт логистику на себя, забирает партию своим или наёмным транспортом из точки продавца)
  - «цена на воротах» (покупатель называет цену на своих воротах, продавец доставляет сырьё самостоятельно)
- цена за килограмм по каждой позиции объявления
- для позиций, по которым покупатель не готов делать предложение, — отметка «не интересует»
- в случае «цены на воротах» — дополнительно поле «город», куда ожидается доставка (только буквы, без адреса)

### 12.2. Цены по нескольким позициям
Если объявление содержит несколько позиций сырья, покупатель обязан по каждой позиции либо указать цену за килограмм, либо проставить отметку «не интересует». Предложение только на часть позиций допускается и не отклоняется системой.

Если покупатель обнаружил в объявлении позицию, которую продавец не указал явно, покупатель может нажать «добавить ещё одну цену» и предложить цену на дополнительную номенклатуру. Факт добавления отображается продавцу в карточке предложения как отдельная строка.

### 12.3. Логистика
Поле условия цены одновременно определяет ответственность за логистику:
- «цена с места» — логистика силами покупателя
- «цена на воротах» — логистика силами продавца

Дополнительное поле «кто везёт» не используется, так как однозначно определяется условием цены.

### 12.4. Детализация доставки
В случае «цены на воротах» покупатель указывает только город назначения. Точный адрес приёмки на стороне покупателя согласовывается по телефону после принятия предложения и не является частью предложения.

### 12.5. Прочие условия
Прочие условия (отсрочка, способ оплаты, требования к качеству при приёмке, график приёмки и иные детали) в первой версии не являются частью формализованного предложения и обсуждаются сторонами по телефону после принятия предложения.

## 13. Ценовое предложение: редактирование и отзыв

### 13.1. Редактирование предложения
Покупатель может в любой момент до принятия продавцом изменить параметры своего предложения:
- повысить или понизить цену по любой из позиций
- изменить условие цены
- изменить город (в случае «цены на воротах»)
- изменить набор позиций, на которые делается предложение

При каждом редактировании продавцу направляется уведомление об изменении предложения.

### 13.2. Отзыв предложения
Покупатель может в любой момент до принятия отозвать своё предложение полностью. При отзыве предложение переводится в статус «отозвано покупателем» и становится недоступным продавцу для принятия. Продавцу направляется уведомление об отзыве.

### 13.3. Невозможность редактирования после принятия
После того как продавец принял предложение (нажал «Принять»), покупатель больше не может его редактировать или отзывать. Все дальнейшие договорённости ведутся по телефону.

### 13.4. Повторное предложение после «Не договорились»
Если продавец принял предложение, получил контакты, по результатам переговоров нажал «Не договорились», соответствующее предложение переходит в финальный статус «не принято». Покупатель после этого может подать новое предложение по тому же объявлению, если объявление ещё активно.
