## 1. Назначение раздела

Настоящий документ фиксирует перечень событий торговой площадки, которые порождают уведомления пользователям. Каналы, шаблоны, пользовательские настройки, правила адресации, очереди и журналирование описывает документ `07-platform/notifications.md`. Настоящий документ перечисляет только события и их получателей в терминах роли в сделке.

## 2. События для покупателя
Получатель — пользователь компании-покупателя, подавшей предложение.
- его предложение принято продавцом, контактные данные раскрыты
- зафиксировано «Договорились» — сделка состоялась
- зафиксировано «Не договорились» по его предложению
- продавец выбрал другого покупателя (сделка состоялась с другим)
- срок объявления истёк, его предложение не было принято
- объявление снято продавцом с указанием причины
- объявление снято модератором
- продавец оставил на него отзыв, отзыв прошёл модерацию
- отзыв на него снят модератором по жалобе

## 3. События для продавца
Получатель — пользователь компании-заготовителя, разместившей объявление.
- поступило новое предложение
- предложение отредактировано покупателем
- предложение отозвано покупателем
- напоминание о необходимости зафиксировать «Договорились» или «Не договорились» по принятому предложению (до истечения двадцати четырёх часов)
- объявление автоматически переведено в архив по причине истечения срока либо со статусом «не актуально»
- напоминание об истечении срока объявления (за несколько дней до окончания)
- на его компанию оставлен новый отзыв, отзыв прошёл модерацию
- объявление снято модератором с указанием причины
- покупатель оставил отзыв по состоявшейся сделке

## 4. Общие служебные события торговой площадки
- одобрение или отклонение отзыва модератором (автору отзыва)
- одобрение или отклонение ответа на отзыв модератором
- решение модератора по поданной жалобе (автору жалобы)

Системные уведомления платформы (изменения правил, обновления, технические работы) принадлежат платформенному уровню и фиксируются в `07-platform/notifications.md`.

## 5. Каналы и настройки
Все события торговой площадки относятся к категории «торговая площадка» в едином каталоге уведомлений и подчиняются её правилам:
- доступные каналы — in-app и email
- категория отключаемая, по умолчанию включена
- SMS для событий торговой площадки не используется

Состав каналов, форму и формулировки шаблонов, правила доставки, повторов и журналирования фиксирует `07-platform/notifications.md`.
