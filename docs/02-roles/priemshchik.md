## 1. Приёмщик

### 1.1. Назначение роли
Приёмщик — это операционная роль внутри компании, предназначенная для фиксации первичных данных по поступлению сырья, весу, сумме операции, форме оплаты и сопутствующим учётным действиям в пределах предоставленного доступа.

### 1.2. Где используется роль
Роль используется только внутри кабинета компании типа:
- заготовитель

### 1.3. Основной уровень доступа
Приёмщик имеет доступ к операционным действиям, связанным с поступлением сырья, фиксацией, расчётом и оформлением первичных данных, но не должен автоматически получать доступ ко всем финансовым, управленческим и системным данным компании.

### 1.4. Доступные разделы
Приёмщику могут быть доступны:
- поступление сырья
- первичный учёт сырья
- остатки сырья и история складских движений
- карточки поставщиков и клиентов в пределах операционной работы
- записи по взвешиванию
- записи по партии сырья
- оформление первичных учётных записей
- связанные документы и печатные формы, если это предусмотрено модулем
- история собственных и связанных операционных действий

### 1.5. Данные, доступные для просмотра
Приёмщик может просматривать:
- данные по поступлению сырья и отгрузке в пределах своей зоны работы
- карточки контрагентов в пределах данных, необходимых для оформления операции
- записи по сырью, весу, объёму, качественным характеристикам и расчётам в пределах предоставленного доступа
- данные по конкретной операции, если это предусмотрено логикой роли
- историю операций, созданных им самим или доступных по настройкам компании

### 1.6. Данные, которые можно создавать
Приёмщик может создавать:
- записи поступления сырья
- записи по взвешиванию
- первичные записи по сырью
- первичные учётные записи
- черновики документов
- данные по расчёту суммы операции на основании веса, цены и прочих утверждённых параметров
- корректировки остатков сырья

### 1.7. Данные, которые можно редактировать
Приёмщик может редактировать:
- собственные черновики и незавершённые операционные записи
- данные по поступлению сырья до момента подтверждения, проведения или закрытия операции
- отдельные технические поля записи, если это допускается бизнес-логикой модуля

### 1.8. Данные, которые можно удалять
Приёмщик может удалять или архивировать:
- собственные черновики
- ошибочно созданные незавершённые записи
- непроведённые учётные записи, если это допускается системой

Проведённые, подтверждённые или влияющие на итоговые расчёты записи удаляться не должны и должны изменяться только через отдельные предусмотренные системой сценарии.

### 1.9. Действия, требующие подтверждения
Для приёмщика подтверждения должны требовать:
- завершение поступления сырья
- фиксация итогового веса или суммы
- отмена уже подтверждённой записи
- корректировка значимых учётных данных после подтверждения операции

### 1.10. Финансовые данные
Приёмщик не имеет доступа к финансовому контуру компании.

Приёмщик видит сумму конкретного поступления сырья и выбранную форму оплаты только как часть операционной формы поступления сырья. Доступ к оплатам, задолженностям, бухгалтерским данным, расчётным данным, финансовым отчётам и финансовой аналитике отсутствует.

### 1.11. Ограничения роли
Приёмщик:
- не должен видеть полную бухгалтерию компании
- не должен видеть стратегическую аналитику компании
- не должен управлять сотрудниками и ролями
- не должен менять системные настройки компании
- не должен получать доступ к административному контуру платформы
- не должен видеть данные других компаний
- не имеет доступа к модулю торговой площадки в первой версии

### 1.12. Особенности роли
- роль может совмещаться с ролью бухгалтера
- роль применяется только внутри кабинета заготовителя в рамках CRM-контура первой версии
- логика операций поступления сырья и первичного учёта должна быть подробно зафиксирована отдельно в документе по CRM и учёту
