## 1. Менеджер по продажам

### 1.1. Назначение роли
Менеджер по продажам — это роль внутри CRM заготовителя, предназначенная для реализации сырья заготовителя трейдерам и переработчикам, ведения покупателей, сопровождения продажных сделок, постановки задач, ведения коммуникаций и управления продажным процессом по этапам.

### 1.2. Где используется роль
Роль используется только внутри CRM-кабинета компании типа:
- заготовитель

### 1.3. Основной уровень доступа
Менеджер по продажам имеет доступ к продажному контуру CRM заготовителя, связанным карточкам покупателей, сделкам, задачам, этапам воронки, рабочим коммерческим расчетам и сопутствующим данным, но не должен автоматически получать доступ к полной бухгалтерии, системным настройкам компании и административному контуру платформы.

### 1.4. Доступные разделы
Менеджеру по продажам могут быть доступны:
- торговая площадка
- CRM компании в части продаж
- карточки покупателей и клиентов
- продажные сделки
- этапы и воронки
- задачи и коммуникации по клиентам
- история взаимодействий
- коммерческие предложения
- рабочие расчеты
- данные по сырью в пределах продажного контура
- часть аналитики по продажам
- остатки сырья и история складских движений
- связанные документы в пределах утвержденной логики модуля
- объявления компании
- поступившие предложения по объявлениям

### 1.5. Данные, доступные для просмотра
Менеджер по продажам может просматривать:
- карточки покупателей
- историю взаимодействий с клиентами
- продажные сделки и их статусы
- этапы и воронки по клиентам и сделкам
- информацию по номенклатуре сырья
- рабочие продажные цены и условия
- собственные и командные задачи по продажам
- текущие остатки сырья и доступный объём

### 1.6. Данные, которые можно создавать
Менеджер по продажам может создавать:
- карточки покупателей
- продажные сделки
- заметки, комментарии и историю коммуникаций
- задачи по клиентам
- предварительные коммерческие расчеты
- черновики коммерческих предложений
- внутренние рабочие записи по продажному процессу
- корректировки остатков сырья
- объявления компании

В рамках торговой площадки менеджер по продажам также может принимать предложения по объявлениям компании и фиксировать по ним результаты «Договорились» и «Не договорились».

### 1.7. Данные, которые можно редактировать
Менеджер по продажам может редактировать:
- карточки покупателей
- собственные и доступные сделки по продажам
- коммуникационные записи
- задачи
- рабочие коммерческие расчеты
- черновики коммерческих предложений
- этапы и статусы воронки в пределах разрешенной логики
- объявления компании

### 1.8. Данные, которые можно удалять
Менеджер по продажам может удалять или архивировать:
- собственные черновики
- коммуникационные записи, если это допускается правилами компании
- незавершенные сделки или черновики сделок, если они не переведены в значимый статус
- задачи и рабочие заметки
- объявления компании

Удаление подтвержденных или проведенных продажных операций должно быть ограничено.

### 1.9. Действия, требующие подтверждения
Для менеджера по продажам подтверждения должны требовать:
- перевод сделки в значимый статус
- согласование коммерческих условий, если это предусмотрено бизнес-процессом
- изменение критичных параметров продажи после согласования
- отмена или архивирование значимых продажных записей
- действия, влияющие на цену или объём в пределах утвержденных сценариев

### 1.10. Финансовые данные
Менеджер по продажам не имеет доступа к финансовому контуру компании.

Менеджер по продажам может видеть рабочие продажные цены и коммерческие условия в продажном контуре, но не видит оплаты, задолженности, бухгалтерские данные, расчётные данные, финансовые отчёты и финансовую аналитику.

### 1.11. Ограничения роли
Менеджер по продажам:
- не должен иметь доступа к полному бухгалтерскому контуру
- не должен управлять сотрудниками и ролями
- не должен менять системные настройки компании
- не должен иметь доступа к административному контуру платформы
- не должен видеть данные других компаний
- не должен получать доступ к тем закупочным и финансовым данным, которые не нужны для его продажной работы

### 1.12. Особенности роли
- роль может совмещаться с ролью руководителя, а также с ролью собственника компании
- внутри одной компании типа «заготовитель» может быть несколько менеджеров по продажам
- один и тот же пользователь может одновременно иметь роль менеджера по продажам и роль менеджера по закупкам
- в рамках торговой площадки объём прав менеджера по продажам идентичен собственнику компании и руководителю
- детализация воронки продаж, этапов сделки и полей карточки покупателя должна быть отдельно зафиксирована в документе по CRM заготовителя
