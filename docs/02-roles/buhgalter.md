## 1. Бухгалтер

### 1.1. Назначение роли
Бухгалтер — это роль, предназначенная для работы с финансовыми, расчетными, учетными и документальными данными компании.

Основная задача роли — фиксировать, проверять, проводить и контролировать финансовые и расчетные операции компании в пределах реализованного функционала системы.

### 1.2. Где используется роль
Роль используется только внутри кабинета компании типа:
- заготовитель

### 1.3. Основной уровень доступа
Бухгалтер имеет доступ к финансовым, расчетным, документальным и связанным учетным данным компании, но не должен автоматически получать полный управленческий доступ ко всем остальным разделам компании.

### 1.4. Доступные разделы
Бухгалтеру могут быть доступны:
- расчетные данные
- бухгалтерские данные
- оплаты
- задолженности
- документы
- карточки контрагентов
- сделки в части, связанной с расчетами и документами
- учет сырья, остатки и история складских движений
- подписка и платные функции компании в части оплаты
- история финансовых операций
- отчетность и финансовая аналитика

### 1.5. Данные, доступные для просмотра
Бухгалтер может просматривать:
- финансовые данные компании
- расчетные данные
- взаиморасчеты с контрагентами
- документы
- задолженности
- оплаты
- связанные со сделками и учетом сведения, необходимые для оформления и проверки операций
- карточки контрагентов в части реквизитов и расчетов

### 1.6. Данные, которые можно создавать
Бухгалтер может создавать:
- финансовые записи
- расчетные записи
- документы
- платежные записи
- записи по задолженности
- корректировки в пределах разрешенных сценариев
- связанные реквизитные данные контрагентов
- учетные записи, необходимые для финансового закрытия операций
- корректировки остатков сырья

### 1.7. Данные, которые можно редактировать
Бухгалтер может редактировать:
- реквизиты контрагентов
- документы
- расчетные записи
- данные по оплатам
- данные по задолженности
- корректировки в пределах разрешенных сценариев
- финансовые комментарии и сопровождающие сведения

### 1.8. Данные, которые можно удалять
Бухгалтер может удалять или архивировать только те записи, удаление которых явно допускается финансовой логикой системы.

По умолчанию:
- проведенные финансовые операции не должны удаляться без специального сценария
- юридически значимые и расчетные записи не должны физически удаляться
- вместо удаления должны использоваться механизмы отмены, сторнирования, аннулирования, архивирования или корректировки, если такие сценарии утверждены отдельно

### 1.9. Действия, требующие подтверждения
Для бухгалтера подтверждения должны требовать:
- проведение финансовых документов
- отмена проведенных операций
- корректировка значимых расчетных данных
- изменение реквизитов, влияющих на расчеты
- действия, влияющие на задолженности, оплаты и финансовые итоги

### 1.10. Финансовые данные
Бухгалтер имеет расширенный доступ к финансовым и расчетным данным своей компании.

При этом:
- доступ к стратегической аналитике может быть ограничен отдельно
- доступ к кадровым данным должен быть ограничен
- доступ к системным настройкам компании должен быть ограничен
- доступ к остаткам сырья и корректировкам остатков предоставляется в соответствии с общими правилами данного документа

### 1.11. Ограничения роли
Бухгалтер:
- не должен автоматически управлять сотрудниками компании
- не должен назначать роли, если такое право не дано отдельно
- не должен иметь доступ к административному контуру платформы
- не должен видеть данные других компаний
- не должен менять системные настройки компании, если это не связано напрямую с финансовым блоком и не разрешено отдельно
- не имеет доступа к модулю торговой площадки в первой версии

### 1.12. Особенности роли
- один пользователь может совмещать роль бухгалтера с другими ролями
- в компании может быть несколько бухгалтеров
- доступ бухгалтера к отдельным разделам определяется утверждёнными спецификациями модулей
