## 1. Назначение раздела
Настоящий документ фиксирует перечень ролей в ЭкоПлатформе, их назначение, границы доступа, допустимые действия и ограничения.

Все права доступа в системе должны реализовываться только на основании данного документа и связанных утвержденных спецификаций модулей.

## 2. Общие принципы модели доступа
- доступ к данным должен предоставляться по роли пользователя
- доступ также должен ограничиваться принадлежностью пользователя к конкретной компании
- один пользователь может иметь одну или несколько ролей внутри своей компании
- финансовый контур компании доступен только собственнику компании и бухгалтеру
- управление сотрудниками и ролями доступно только собственнику компании, если иное явно не зафиксировано в утверждённой спецификации
- доступ к административным функциям платформы должен быть полностью отделен от доступа пользователей компаний
- права доступа должны учитывать не только просмотр, но и создание, редактирование, удаление, публикацию, подтверждение, проведение и модерацию данных

## 3. Основные типы субъектов доступа
В системе существуют следующие уровни субъектов доступа:
- платформа
- компания
- пользователь внутри компании

## 4. Основные типы компаний
- заготовитель
- трейдер
- переработчик

## 5. Пользователи внутри компании
Состав ролей внутри компании зависит от типа компании.

Роли внутри кабинета компании типа «заготовитель»:
- собственник компании
- руководитель
- бухгалтер
- приёмщик
- менеджер по закупкам
- менеджер по продажам

Роли внутри кабинета компании типа «трейдер»:
- собственник компании
- доверенное лицо

Роли внутри кабинета компании типа «переработчик»:
- собственник компании
- доверенное лицо

Роли руководителя, бухгалтера, приёмщика, менеджера по закупкам и менеджера по продажам применяются только внутри кабинета заготовителя.

Роль доверенного лица применяется только внутри кабинетов трейдера и переработчика.

У трейдера и переработчика CRM-контура в первой версии нет.

## 6. Пользователи со стороны платформы
- контент-менеджер
- модератор
- администратор платформы

## 7. Принципы назначения ролей
- один пользователь может совмещать несколько ролей
- права пользователя определяются совокупностью назначенных ролей
- при конфликте прав должен применяться наиболее ограничительный сценарий, если иное не зафиксировано отдельно
- каждая роль должна иметь четко определенный набор разрешенных действий
- роли платформы не должны смешиваться с ролями внутри компаний

## 8. Что должно быть зафиксировано по каждой роли далее
Для каждой роли далее должны быть отдельно описаны:
- назначение роли
- где используется роль
- к каким разделам есть доступ
- какие данные доступны для просмотра
- какие данные можно создавать
- какие данные можно редактировать
- какие данные можно удалять
- какие действия требуют подтверждения
- какие финансовые данные скрыты
- какие ограничения действуют для роли

## 10. Общие правила фиксации прав доступа
Если какое-либо право, ограничение или сценарий не описан явно, он не считается утвержденным и не должен реализовываться по умолчанию.

Доступ к финансовому контуру компании имеют только:
- собственник компании
- бухгалтер

К финансовому контуру относятся оплаты, задолженности, бухгалтерские данные, расчётные данные, финансовые отчёты и финансовая аналитика.

Доступ к остаткам сырья и истории складских движений в CRM заготовителя имеют:
- собственник компании
- бухгалтер
- руководитель
- приёмщик
- менеджер по закупкам
- менеджер по продажам

Корректировки остатков сырья в CRM заготовителя могут вносить:
- собственник компании
- бухгалтер
- руководитель
- приёмщик
- менеджер по закупкам
- менеджер по продажам

Оформлять поступление сырья в CRM заготовителя могут:
- собственник компании
- руководитель
- бухгалтер
- приёмщик
- менеджер по закупкам
- менеджер по продажам

### 6.6. Ролевая модель первой версии
В первой версии кабинета заготовителя используются роли:
- собственник
- руководитель
- бухгалтер
- приёмщик
- менеджер по закупкам
- менеджер по продажам

Отдельной роли «универсальный менеджер» в проекте нет.

Один пользователь может одновременно иметь:
- роль менеджера по закупкам
- роль менеджера по продажам

Оформлять поступление сырья могут все шесть ролей кабинета заготовителя (собственник, руководитель, бухгалтер, приёмщик, менеджер по закупкам, менеджер по продажам).

Доступ к остаткам и корректировкам остатков имеют:
- собственник
- бухгалтер
- руководитель
- приёмщик
- менеджер по закупкам
- менеджер по продажам

### 7.2. Пользователи внутри компаний
Пользователи внутри компании делятся на две группы.

Общие пользователи компании:
- собственник компании
- руководитель
- бухгалтер

Пользователи CRM-контура заготовителя:
- приёмщик
- менеджер по закупкам
- менеджер по продажам

CRM-роли используются только внутри кабинета заготовителя.
