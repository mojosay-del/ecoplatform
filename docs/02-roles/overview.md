## 1. Назначение раздела
Настоящий документ фиксирует перечень платформенных ролей в ЭкоПлатформе, их назначение, границы доступа, допустимые действия и ограничения.

Все права доступа в системе должны реализовываться только на основании данного документа и связанных утверждённых спецификаций модулей.

## 2. Общие принципы модели доступа
- доступ к данным компании ограничивается принадлежностью пользователя к этой компании
- каждая компания на платформе имеет один пользовательский аккаунт; внутренних ролей внутри компании в первой версии нет
- доступ к административным функциям платформы должен быть полностью отделён от доступа пользователей компаний
- права доступа должны учитывать не только просмотр, но и создание, редактирование, удаление, публикацию, подтверждение и модерацию данных

## 3. Основные типы субъектов доступа
В системе существуют следующие уровни субъектов доступа:
- платформа (платформенные сотрудники)
- компания (один пользователь на компанию)

## 4. Основные типы компаний
- заготовитель
- трейдер
- переработчик

Каждая компания на платформе представлена единственным пользовательским аккаунтом. Этот аккаунт получает доступ ко всем модулям, доступным её типу компании, в пределах оплаченной подписки.

## 5. Платформенные роли
- контент-менеджер
- модератор
- администратор платформы

## 6. Принципы назначения платформенных ролей
- один платформенный сотрудник может совмещать несколько ролей
- права сотрудника определяются совокупностью назначенных ролей
- при конфликте прав применяется наиболее ограничительный сценарий, если иное не зафиксировано отдельно
- каждая роль должна иметь чётко определённый набор разрешённых действий
- платформенные роли не пересекаются с пользовательскими аккаунтами компаний

## 7. Что должно быть зафиксировано по каждой роли далее
Для каждой платформенной роли далее должны быть отдельно описаны:
- назначение роли
- где используется роль
- к каким разделам есть доступ
- какие данные доступны для просмотра
- какие данные можно создавать
- какие данные можно редактировать
- какие данные можно удалять
- какие действия требуют подтверждения
- какие ограничения действуют для роли

## 8. Перечень ролей
В этом документе описываются:
1. контент-менеджер
2. модератор
3. администратор платформы

## 9. Общие правила фиксации прав доступа
Если какое-либо право, ограничение или сценарий не описан явно, он не считается утверждённым и не должен реализовываться по умолчанию.
