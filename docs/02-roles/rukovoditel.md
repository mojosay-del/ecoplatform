## 1. Руководитель

### 1.1. Назначение роли
Руководитель — это управленческая роль внутри компании, предназначенная для контроля операционной деятельности, сотрудников, сделок, показателей и рабочих процессов без обязательного доступа ко всем критичным настройкам собственника.

### 1.2. Где используется роль
Роль используется только внутри кабинета компании типа:
- заготовитель

### 1.3. Основной уровень доступа
Руководитель имеет расширенный управленческий доступ к рабочим процессам своей компании, но его права ограничены по сравнению с собственником в части критичных юридических и системных действий.

### 1.4. Доступные разделы
Руководителю могут быть доступны:
- торговая площадка
- CRM компании — только для компании типа «заготовитель»
- сделки
- контрагенты
- карточки клиентов, поставщиков и покупателей
- учет сырья
- складские и операционные данные
- объявления компании
- поступившие предложения по объявлениям
- операционная аналитика компании без финансовых данных
- рейтинги, отзывы и репутационные данные компании
- сотрудники компании
- задачи и рабочие процессы

### 1.5. Данные, доступные для просмотра
Руководитель может просматривать:
- основную операционную деятельность компании
- карточки контрагентов
- сделки и историю работы по ним
- данные по сотрудникам в пределах рабочих процессов
- производственные, складские и закупочные данные
- операционные отчеты компании без финансовых данных

### 1.6. Данные, которые можно создавать
Руководитель может создавать:
- сделки
- задачи
- карточки контрагентов
- внутренние рабочие записи
- объявления компании
- часть записей по учету сырья, если это разрешено модулем
- назначения задач сотрудникам

Руководитель также может принимать предложения по объявлениям компании и фиксировать по ним результаты «Договорились» и «Не договорились».

### 1.7. Данные, которые можно редактировать
Руководитель может редактировать:
- сделки
- карточки контрагентов
- объявления компании
- задачи
- часть операционных и складских данных
- часть данных по сотрудникам в пределах управленческого контура
- отдельные параметры рабочих процессов компании

### 1.8. Данные, которые можно удалять
Руководитель может удалять или архивировать:
- черновики
- задачи
- объявления компании
- часть внутренних записей, если это допускается модулем
- карточки контрагентов, если они не участвуют в активных или проведенных операциях

Удаление данных, влияющих на финансы, бухгалтерию, юридически значимые операции или итоговую отчетность, должно быть ограничено.

### 1.9. Действия, требующие подтверждения
Подтверждения должны требовать:
- удаление критичных рабочих записей
- архивирование или блокировка значимых сущностей

### 1.10. Финансовые данные
Руководитель не имеет доступа к финансовому контуру компании.

Руководитель может видеть операционные данные, необходимые для управления работой, но не видит оплаты, задолженности, бухгалтерские данные, расчётные данные, финансовые отчёты и финансовую аналитику.

### 1.11. Ограничения роли
Руководитель:
- не должен иметь безусловный доступ к системным настройкам компании уровня собственника
- не должен иметь доступ к бухгалтерским и финансовым данным
- не должен иметь доступ к административному контуру платформы
- не должен видеть данные других компаний

### 1.12. Особенности роли
- роль руководителя может совмещаться с другими ролями
- внутри одной компании может быть несколько руководителей
- конкретный набор операционных полномочий руководителя фиксируется утверждёнными спецификациями модулей
