## 1. Модератор

### 1.1. Назначение роли
Модератор — это платформенная роль, предназначенная для контроля качества пользовательского контента, объявлений, отзывов, комментариев, карточек участников рынка и иных материалов, подлежащих проверке в рамках правил платформы.

### 1.2. Где используется роль
Роль используется на уровне платформы и не относится к внутренним ролям компаний.

### 1.3. Основной уровень доступа
Модератор имеет доступ к контуру проверки и ограничения пользовательского контента, но не должен автоматически получать полный административный доступ ко всей системе.

### 1.4. Доступные разделы
Модератору могут быть доступны:
- объявления
- карточки компаний в части публичных данных
- отзывы
- комментарии
- обсуждения
- пользовательский контент
- жалобы и обращения на контент
- очередь модерации
- история решений по модерации
- внутренние заметки по модерации

### 1.5. Данные, доступные для просмотра
Модератор может просматривать:
- пользовательский и публичный контент, подлежащий проверке
- жалобы, причины жалоб и связанные материалы
- историю решений по конкретным сущностям
- публичные данные карточек компаний и объявлений
- связанные комментарии, отзывы, обсуждения и вложения в пределах модерационного сценария

### 1.6. Данные, которые можно создавать
Модератор может создавать:
- решения по модерации
- служебные комментарии по кейсам
- внутренние заметки
- статусы проверки
- запросы на доработку или ограничение контента
- причины отклонения, скрытия или блокировки в пределах утвержденной логики

### 1.7. Данные, которые можно редактировать
Модератор может редактировать:
- статусы модерации
- внутренние комментарии
- причины решений
- параметры ограничений в пределах утвержденной логики
- отдельные поля модерационной сущности, если это предусмотрено системой

### 1.8. Данные, которые можно удалять
Модератор может удалять или архивировать:
- внутренние модерационные заметки
- служебные черновики решений
- контент только в пределах отдельного утвержденного сценария модерации

По умолчанию:
- удаление пользовательского контента должно заменяться скрытием, блокировкой, снятием с публикации, отклонением или архивированием, если иное не зафиксировано отдельно
- физическое удаление значимых пользовательских материалов должно быть ограничено

### 1.9. Действия, требующие подтверждения
Для модератора подтверждения должны требовать:
- блокировка или скрытие контента
- отклонение объявлений или карточек
- ограничение публичной видимости сущности
- повторная публикация ранее ограниченного материала
- эскалация спорного случая на уровень администратора

### 1.10. Финансовые данные
Модератор не должен иметь доступа к финансовым данным компаний.

По умолчанию:
- доступ к внутренней бухгалтерии компаний отсутствует
- доступ к финансовому контуру платформы отсутствует
- доступ к платежным деталям, подпискам и расчетам отсутствует, если иное не требуется отдельным утвержденным сценарием

### 1.11. Ограничения роли
Модератор:
- не должен управлять сотрудниками компаний и их ролями
- не должен видеть внутренние непубличные данные компаний, если это не требуется отдельным модерационным сценарием
- не должен иметь полный доступ к системным настройкам платформы
- не должен выполнять административные действия вне контура модерации без отдельной роли администратора платформы

### 1.12. Особенности роли
- один пользователь платформы может совмещать роль модератора с ролью контент-менеджера или администратора, если это утверждено отдельно
- логика жалоб, очереди модерации, статусов, причин решений, ограничений и эскалации должна быть отдельно подробно зафиксирована в документе по модерации и пользовательскому контенту
