## 1. Назначение раздела
Настоящий документ фиксирует устройство публичного интерфейса модуля «Новости» — экранов, которые видит пользовательский аккаунт компании в кабинете платформы.

Документ продолжает [overview.md](./overview.md), раздел 11, и развивает каждый из публичных экранов до уровня состава блоков и поведения.

Документ определяет:
- точку входа в модуль из бокового меню кабинета;
- ленту новостей с масонри-сеткой;
- фильтр по тегам;
- страницу отдельной новости (модалка из ленты + отдельная страница по прямой ссылке);
- работу с комментариями и ответами в публичном интерфейсе;
- работу с лайками новости и комментариев;
- плеер аудио;
- поведение в состояниях компании;
- поведение при ошибках встраивания видео и недоступности аудио.

## 2. Точка входа
В боковом меню кабинета пользовательского аккаунта компании присутствует пункт «Новости» в разделе «Главная». При нажатии открывается лента новостей (раздел 3).

Точка входа отображается всем типам компаний (заготовитель, трейдер, переработчик) с одинаковым составом модуля.

## 3. Лента новостей

### 3.1. Состав экрана
Лента отображает все новости в состоянии `published`. Новости в `draft` пользователю не видны.

Экран состоит из:
- заголовка раздела;
- фильтра по тегам в верхней части (раздел 4);
- ленты карточек новостей в виде масонри-сетки.

Сортировка по умолчанию — по моменту первой публикации, новые сверху. Альтернативные сортировки в первой версии не предусмотрены.

### 3.2. Состав карточки новости
- обложка (`coverImage`), если задана. Если обложка не задана, карточка отображается без изображения, в одну колонку текста;
- заголовок (с инлайн-форматированием);
- лид (`lead`);
- список тегов в виде чипов (нажатие на чип фильтрует ленту по этому тегу — раздел 4);
- относительное время публикации («8 минут назад», «3 часа назад», «вчера», далее точная дата);
- счётчик лайков и кнопка «Лайк» (раздел 6.1);
- счётчик комментариев.

Если в новости первый блок — `audio`, на карточке дополнительно отображается компактный аудио-плеер со ссылкой на полный плеер на странице новости. Это позволяет пользователю прослушать подкаст прямо из ленты, не открывая страницу.

Аналогично, если первый блок — `video`, на карточке отображается превью видео с кнопкой play. Клик по превью открывает страницу новости с автозапуском видео (поведение автозапуска определяется самим Rutube).

### 3.3. Масонри-сетка
- На широких экранах — несколько колонок (3–4 в зависимости от ширины);
- На средних — 2 колонки;
- На мобильных — одна колонка.

Конкретные брейкпоинты — на стороне дизайн-системы.

### 3.4. Бесконечный скролл
Лента подгружает новости порциями по мере прокрутки страницы вниз. Размер порции — параметр конфигурации платформы (рекомендуемое значение — 20 новостей на страницу).

При достижении конца ленты (нет более старых новостей) отображается служебная плашка «Это все новости».

### 3.5. Поведение ленты в состояниях компании
- `active`, `past_due`, `demo` — лента доступна;
- `suspended`, `archived`, `blocked` — лента недоступна.

## 4. Фильтр по тегам

### 4.1. Размещение
Над лентой отображается панель фильтра по тегам:
- горизонтальный скроллируемый список чипов с самыми популярными тегами (по `usageCount`);
- кнопка «Все теги» — открывает выпадающий список со всеми тегами и поиском по названию.

### 4.2. Применение фильтра
Пользователь нажимает на чип тега — лента фильтруется до новостей, имеющих этот тег. Чип становится активным (выделенным).

При нажатии на ещё один чип — фильтр расширяется по принципу пересечения: лента отображает новости, имеющие **все** выбранные теги. Это даёт сужение выборки.

### 4.3. Снятие фильтра
- нажатие на активный чип повторно — снимает фильтр по этому тегу;
- кнопка «Очистить» в панели фильтра — снимает все активные фильтры одним действием.

### 4.4. URL и прямые ссылки
Активные фильтры отражаются в URL ленты как query-параметры: `/news?tag=oborudovanie&tag=zakupki`. Прямой переход по такому URL открывает ленту с применёнными фильтрами.

### 4.5. Кликабельность тегов на карточке
Теги, отображаемые на карточке новости в ленте, также кликабельны и применяют соответствующий фильтр.

## 5. Страница отдельной новости

### 5.1. Два режима отображения
Страница отдельной новости открывается в двух режимах:
- **Модалка** — открывается при клике на карточку из ленты. Поверх затемнённого фона ленты, с кнопкой закрытия, без перезагрузки страницы. Сохраняет позицию пользователя в ленте;
- **Отдельная страница** — открывается при прямом переходе по ссылке `/news/{slug}`. Полноценная страница с собственным URL.

Оба режима полностью эквивалентны по содержимому. Различие — только в обвязке (модалка versus полная страница).

### 5.2. Состав экрана
Содержимое экрана идентично в обоих режимах:
- заголовок новости (с инлайн-форматированием);
- список тегов в виде чипов (кликабельны — открывают ленту с фильтром по тегу);
- относительное время публикации;
- последовательность блоков контента (раздел 5.3);
- кнопка лайка с счётчиком;
- кнопка «Скопировать ссылку» — копирует прямую ссылку на новость в буфер обмена для отправки коллеге внутри платформы или сохранения. Внешний шеринг (Telegram, ВК, прочие) в первой версии не реализуется (см. `overview.md`, раздел 4);
- секция комментариев (раздел 6).

### 5.3. Отрисовка блоков контента
В основном блоке последовательно отображаются блоки контента новости (`NewsContentBlock`) в порядке, заданном `position`. Отрисовка по типам:

- `heading` — заголовок второго уровня;
- `subheading` — заголовок третьего уровня;
- `paragraph` — отрисовка markdown: жирный, курсив, инлайн-ссылки, маркированные и нумерованные списки;
- `image` — изображение по `fileId` с подписью `caption` под ним и `altText` как атрибут доступности;
- `gallery` — сетка миниатюр; клик по миниатюре открывает изображение в полноэкранном просмотре (lightbox) с возможностью переключения между изображениями;
- `video` — встроенный плеер Rutube с использованием `rutubeUrl`;
- `audio` — стандартный плеер платформы (раздел 7).

Между блоками сохраняется типовой вертикальный отступ. Дополнительное оформление (фоны, рамки) в первой версии не применяется.

### 5.4. Кнопка закрытия модалки
В правом верхнем углу модалки — крестик закрытия. Клик возвращает пользователя в ленту в ту же позицию, где он находился до открытия новости. URL не меняется.

Альтернативно: клик по затемнённому фону за пределами модалки также закрывает её.

### 5.5. Открытие модалки в новом окне
В модалке доступна кнопка «Открыть в новом окне» — переводит пользователя на отдельную страницу `/news/{slug}` (раздел 5.1, режим отдельной страницы) в том же или новом окне. Используется для удобной навигации, шеринга ссылки коллегам.

### 5.6. Поведение страницы при отсутствии новости
Если по прямой ссылке `/news/{slug}` запрашивается слаг, которого нет в системе (новость никогда не существовала или удалена), страница возвращает HTTP 404 с пояснительной плашкой «Новость не найдена» и кнопкой «Вернуться к ленте».

Если запрашивается слаг, который ранее существовал, но был переименован администратором (раздел 8.3 [cabinet-content-manager.md](./cabinet-content-manager.md)), пользователь автоматически перенаправляется на актуальный слаг (HTTP 301).

Если запрашивается слаг новости в состоянии `draft` или удалённой администратором — страница возвращает HTTP 404. Платформа не раскрывает факт существования черновика.

### 5.7. Поведение в состояниях компании
- `active`, `past_due`, `demo` — страница новости доступна;
- `suspended`, `archived`, `blocked` — страница недоступна; пользователь видит сообщение об ограничении доступа со ссылкой на раздел биллинга.

## 6. Лайки и комментарии в публичном интерфейсе

### 6.1. Кнопка лайка
Под основным контентом новости и на карточке в ленте отображается кнопка лайка с счётчиком. Поведение:
- если пользователь не ставил лайк — кнопка нейтральная, клик создаёт `NewsLike`;
- если уже ставил — кнопка отображается как активная (закрашенная), клик удаляет `NewsLike`;
- счётчик обновляется немедленно, без перезагрузки страницы.

### 6.2. Секция комментариев
Под кнопкой лайка отображается секция комментариев:
- заголовок «Комментарии (N)», где N — общее число опубликованных комментариев и ответов;
- форма ввода нового комментария (раздел 6.3);
- список комментариев первого уровня в хронологическом порядке (новые сверху);
- под каждым комментарием первого уровня — кнопка «Ответить» и список его ответов в обратном хронологическом порядке (новые снизу — раздел 4.3 [comments.md](./comments.md)).

### 6.3. Форма ввода комментария
В нижней части секции комментариев (или в нижней панели модалки) — фиксированная форма ввода:
- многострочное текстовое поле с плейсхолдером «Напишите комментарий…»;
- кнопка «Прикрепить» (иконка скрепки) — открывает выбор файлов изображений (до трёх; см. раздел 5 [comments.md](./comments.md));
- кнопка «Отправить».

Если пользователь нажал «Ответить» на конкретном комментарии, форма открывается под этим комментарием с пометкой «В ответ на: {имя автора, первые слова комментария}» и кнопкой отмены ответа. Платформа автоматически устанавливает `parentCommentId`.

### 6.4. Карточка комментария
Каждая карточка комментария содержит:
- имя автора (отображаемое имя пользовательского аккаунта компании);
- название компании автора;
- относительное время публикации;
- текст комментария;
- сетку миниатюр вложений (если есть);
- кнопку лайка с счётчиком;
- кнопку «Ответить» — открывает форму ввода ответа (только для комментариев первого уровня);
- кнопку «Пожаловаться» — открывает диалог жалобы по правилам `07-platform/moderation.md`.

Аватары пользователей отображаются по правилам `07-platform/storage.md`, раздел 5.2.

### 6.5. Отображение скрытых комментариев
Комментарии в состоянии `hidden_by_moderator` (см. [comments.md](./comments.md), раздел 2.3) отображаются как:
- для всех пользователей кроме автора — плашка «Комментарий скрыт модератором»;
- для автора комментария — плашка «Комментарий скрыт модератором» с возможностью раскрыть исходный текст под кат.

Комментарии в состоянии `removed_by_admin` не отображаются никому.

### 6.6. Загрузка комментариев
Если под новостью много комментариев, лента комментариев подгружается порциями. Размер порции — параметр конфигурации платформы (рекомендуемое значение — 20 комментариев на загрузку).

Кнопка «Показать ещё ?» в конце списка подгружает следующую порцию (где «?» — число оставшихся комментариев).

## 7. Плеер аудио

### 7.1. Состав плеера
Стандартный плеер платформы для блока `audio`:
- кнопка play/pause;
- название выпуска подкаста (`episodeTitle`), если задано;
- прогресс-бар с возможностью перемотки;
- индикатор текущего времени и общей длительности;
- регулятор громкости.

Конкретные параметры плеера и его UX — на стороне дизайн-системы.

### 7.2. Поведение при недоступности файла
Если аудиофайл удалён из хранилища или временно недоступен, на месте плеера отображается сообщение «Аудио временно недоступно». Остальные блоки новости остаются доступны.

### 7.3. Воспроизведение из ленты
Если первый блок новости — `audio`, на карточке в ленте отображается компактный плеер (раздел 3.2). Клик на play в карточке начинает воспроизведение прямо в ленте, не открывая страницу новости.

Одновременное воспроизведение нескольких аудио (например, если пользователь нажал play на двух разных карточках) — открытый вопрос для дизайн-системы. Рекомендуемое поведение: при начале воспроизведения нового аудио прежнее автоматически останавливается.

## 8. Поведение при ошибках видео

### 8.1. Ролик Rutube недоступен
Если конкретный ролик Rutube возвращает ошибку (удалён, закрыт автором, ограничен по геолокации), на месте блока `video` отображается плашка «Видео временно недоступно» с указанием подписи `caption`, если она была задана. Остальные блоки новости остаются доступны.

### 8.2. Rutube как сервис недоступен
При массовой недоступности Rutube все блоки `video` всех новостей отображают плашку «Видео временно недоступно». Остальной контент новостей остаётся доступен.

## 9. Поведение при потере доступа в процессе чтения

Если в процессе чтения новости подписка пользователя истекла и компания перешла в `suspended` или иное состояние с закрытым доступом, на следующее обновление страницы пользователь перенаправляется на главную страницу кабинета с уведомлением об ограничении доступа.

В рамках текущей сессии до обновления страницы возможно дочитать открытую новость без перезагрузки. Платформа не выполняет агрессивный middleware-перехват в активной сессии.

## 10. Открытые вопросы первой версии

### 10.1. Параметры таймаута на загрузку embed Rutube
Параметры таймаута на загрузку embed-фрейма Rutube и поведение при долгой загрузке (показ спиннера, плашки об ошибке после N секунд) — открытый вопрос. Конкретное решение фиксируется в техническом задании на интеграцию с Rutube.

### 10.2. Адаптивность mobile
Подробная адаптация ленты, модалки, формы ввода комментария под мобильные экраны (точные брейкпоинты, поведение раскрывающихся секций) — на стороне дизайн-системы.

### 10.3. Поведение нескольких аудио одновременно
Поведение при попытке воспроизведения нескольких аудио одновременно (на разных карточках или одновременно на карточке и в открытой новости) — раздел 7.3, открытый вопрос.

### 10.4. Сортировка комментариев
Альтернативные сортировки комментариев (по числу лайков, по релевантности) в первой версии не поддерживаются. Только хронологический порядок (см. [comments.md](./comments.md), раздел 4.3).

### 10.5. Уведомления об ответах в реальном времени
Live-обновления секции комментариев (новый комментарий появляется без перезагрузки страницы у других пользователей) в первой версии не реализуются. Пользователь видит новые комментарии только после явного обновления страницы.
