## 1. Назначение раздела
Настоящий документ фиксирует поведение прогресса пользовательского аккаунта компании по урокам и обучающим модулям. Документ продолжает [overview.md](./overview.md), раздел 8.

Документ определяет:
- состав сущности `LessonProgress`;
- сценарий ручной отметки урока как пройденного;
- расчёт прогресса по главе и по модулю;
- поведение прогресса при изменении содержимого урока, добавлении нового урока, удалении урока, удалении модуля;
- поведение прогресса в состояниях компании, отличных от `active`;
- ограничения первой версии.

## 2. LessonProgress — отметка пользователя по уроку

### 2.1. Назначение
Запись о факте того, что конкретный пользовательский аккаунт компании отметил конкретный урок как пройденный. Первичный артефакт прогресса.

### 2.2. Состав полей
- `id` — идентификатор;
- `userId` — идентификатор пользовательского аккаунта компании;
- `lessonId` — идентификатор урока;
- `completedAt` — момент отметки урока как пройденного;
- `createdAt` — момент создания записи (равен `completedAt` при первой отметке).

В первой версии запись `LessonProgress` существует только при условии «урок пройден». Промежуточные состояния («просмотрен на 50%», «начат, не завершён») не моделируются — отметка строго бинарная.

### 2.3. Уникальность
Сочетание (`userId`, `lessonId`) уникально. Повторная отметка одного урока одним пользователем не создаёт новую запись.

## 3. Сценарий отметки урока как пройденного

### 3.1. Кнопка «Отметить как пройденный»
На странице урока (см. [public-ui.md](./public-ui.md), раздел 5) пользователь видит кнопку «Отметить как пройденный». Кнопка доступна, если:
- модуль урока в состоянии `published`;
- урок в состоянии `published`;
- пользовательский аккаунт компании имеет доступ к содержимому модуля (`overview.md`, раздел 7.2);
- состояние компании пользователя — `active`, `demo` или `past_due` (раздел 6 настоящего документа);
- запись `LessonProgress` для пары (`userId`, `lessonId`) ещё не существует.

При нажатии кнопки создаётся запись `LessonProgress` с текущим временем в полях `completedAt` и `createdAt`. Кнопка превращается в неактивный индикатор «Урок пройден» с датой прохождения.

### 3.2. Снятие отметки
В первой версии пользователь не может снять отметку «урок пройден». Запись `LessonProgress` создаётся однократно и не удаляется самим пользователем.

Удаление записи происходит только каскадно при удалении урока, главы или модуля (см. [lessons.md](./lessons.md), раздел 10).

Возможность пользователю самостоятельно сбросить прогресс по уроку — открытый вопрос для последующих версий.

### 3.3. Журналирование
Создание записи `LessonProgress` не фиксируется в `AdminActionLog`. Журнал прогресса не предусматривает административный доступ — данные прогресса видит только сам пользователь и используются они только для расчёта показателя в публичном интерфейсе.

## 4. Расчёт прогресса по главе

Прогресс по главе — производный показатель: число опубликованных уроков главы, по которым у пользователя есть запись `LessonProgress`, делённое на общее число опубликованных уроков главы.

В первой версии прогресс по главе отдельно в публичном интерфейсе не отображается (см. [public-ui.md](./public-ui.md), раздел 4 — на странице модуля показывается прогресс модуля целиком). Промежуточный показатель главы используется только как часть расчёта прогресса модуля.

## 5. Расчёт прогресса по модулю

### 5.1. Формула
Прогресс по обучающему модулю — производный показатель:

```
progress = completedLessons / publishedLessons
```

где:
- `completedLessons` — число опубликованных уроков модуля, по которым у пользователя есть запись `LessonProgress`;
- `publishedLessons` — общее число опубликованных уроков модуля (сумма по всем опубликованным главам).

Уроки в состоянии `draft` в расчёте не участвуют ни в числителе, ни в знаменателе.

Главы в расчёте напрямую не участвуют — главы являются группирующим элементом, не влияющим на формулу.

### 5.2. Отображение
Прогресс отображается в двух представлениях:
- доля «X из N», где `X = completedLessons`, `N = publishedLessons`;
- процентное значение `(X / N) × 100`, округлённое до целого.

При `publishedLessons = 0` (модуль опубликован, но все его уроки в `draft`) прогресс не рассчитывается — отображается «—» вместо процентного значения; кнопка «Перейти к обучению» на странице модуля не показывается.

### 5.3. Места отображения
Прогресс модуля показывается:
- на карточке модуля в каталоге (только если у пользователя есть доступ к содержимому);
- на странице обучающего модуля рядом с заголовком и в верхней части оглавления;
- на странице урока в боковой панели (см. [public-ui.md](./public-ui.md), раздел 5.2).

На странице превью модуля (для пользователей без доступа) прогресс не отображается, поскольку отметка прогресса невозможна без открытия уроков.

## 6. Поведение в состояниях компании

### 6.1. `active`
Полный сценарий прогресса. Пользователь отмечает уроки как пройденные, прогресс рассчитывается и отображается.

### 6.2. `demo`
Доступ к содержимому модуля ограничен модулями уровня `basic` (см. `overview.md`, раздел 7.5). По доступным модулям пользователь может отмечать уроки и видеть прогресс. Записи `LessonProgress`, созданные в `demo`, сохраняются при переходе компании в `active`.

### 6.3. `past_due`
Доступ соответствует действующему тарифу. Поведение прогресса идентично состоянию `active`.

### 6.4. `suspended`, `archived`, `blocked`
Доступ к функциональным разделам платформы закрыт. Страницы модулей и уроков недоступны. Кнопка «Отметить как пройденный» недоступна. Существующие записи `LessonProgress` сохраняются и становятся видны при возврате компании в `active`.

## 7. Поведение прогресса при изменениях контента

### 7.1. Изменение содержимого урока
Если контент-менеджер изменил тело урока (добавил, удалил или изменил блоки контента), записи `LessonProgress` по этому уроку **не сбрасываются**. Пользователь, ранее отметивший урок как пройденный, сохраняет отметку. Урок остаётся «пройденным» с прежним `completedAt`.

Это сознательный компромисс: ручная отметка фиксирует факт прохождения, а не привязку к конкретной редакции содержимого. Версионирование редакций урока в первой версии не реализуется (см. `overview.md`, раздел 13).

### 7.2. Добавление нового урока в модуль
При публикации нового урока в опубликованном модуле общее число опубликованных уроков увеличивается. Прогресс пользователя автоматически пересчитывается по формуле раздела 5.1: знаменатель растёт на единицу, числитель не меняется (новый урок ещё не пройден).

В результате процентное значение прогресса у пользователей, ранее прошедших весь модуль, может снизиться (например, со 100% до 95%). На странице модуля у такого пользователя появляется новый урок с пометкой «Не пройден».

### 7.3. Снятие урока с публикации
Если опубликованный урок переведён администратором обратно в `draft`, общее число опубликованных уроков уменьшается. Прогресс пересчитывается. Запись `LessonProgress` пользователей по этому уроку сохраняется, но в расчёт прогресса не включается, пока урок снова не опубликован.

### 7.4. Удаление урока
При удалении урока (см. [lessons.md](./lessons.md), раздел 10.1) каскадно удаляются все записи `LessonProgress` по этому уроку. Знаменатель прогресса модуля уменьшается на единицу.

### 7.5. Удаление главы
При удалении главы каскадно удаляются её уроки и записи прогресса по ним.

### 7.6. Удаление модуля
При удалении модуля каскадно удаляются все главы, уроки и записи прогресса по урокам этого модуля.

## 8. Видимость прогресса

### 8.1. Кому виден прогресс
Прогресс по конкретному уроку и по конкретному модулю виден только тому пользовательскому аккаунту компании, который этот прогресс создал.

### 8.2. Платформенные сотрудники
Администратор платформы и контент-менеджер не имеют доступа к индивидуальным записям прогресса пользователей в первой версии. Аналитика по обучению (например, «процент компаний, прошедших данный модуль») в первой версии не реализуется (см. `07-platform/admin-panel.md`, раздел 19.5).

## 9. Ограничения первой версии
- Отметка прогресса строго ручная; автоматический трекинг по событиям видеоплеера или скроллу страницы не реализуется (см. `overview.md`, раздел 4).
- Прогресс хранится как отметка факта прохождения; промежуточные состояния («50% урока просмотрено») не моделируются.
- Снятие пользователем отметки прогресса в первой версии не предусматривается (раздел 3.2).
- Аналитика прогресса для платформенных сотрудников в первой версии отсутствует (раздел 8.2).
- Линейная блокировка уроков («следующий урок открывается только после прохождения предыдущего») не применяется. Пользователь свободно перемещается между уроками доступного модуля.
- Сертификаты, бейджи, отметки «модуль завершён» сверх отображения 100% не реализуются.
