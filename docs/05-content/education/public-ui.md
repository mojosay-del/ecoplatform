## 1. Назначение раздела
Настоящий документ фиксирует устройство публичного интерфейса модуля «Обучение» — экранов, которые видит пользовательский аккаунт компании в кабинете платформы.

Документ продолжает [overview.md](./overview.md), раздел 10, и развивает каждый из четырёх публичных экранов до уровня состава блоков и поведения.

Документ определяет:
- точку входа в модуль из бокового меню кабинета;
- каталог обучающих модулей;
- страницу превью модуля (для пользователей без доступа);
- страницу обучающего модуля (для пользователей с доступом);
- страницу урока;
- поведение в состояниях компании;
- поведение при ошибках встраивания видео.

## 2. Точка входа
В боковом меню кабинета пользовательского аккаунта компании присутствует пункт «Обучение» в разделе «База знаний». При нажатии открывается каталог обучающих модулей (раздел 3).

Точка входа отображается всем типам компаний (заготовитель, трейдер, переработчик) с одинаковым составом модуля.

## 3. Каталог обучающих модулей

### 3.1. Состав экрана
Каталог отображает все обучающие модули в состоянии `published`. Модули в `draft` пользователю не видны.

Экран состоит из:
- заголовка раздела;
- списка карточек модулей в виде сетки;
- фильтра по уровню платности (`basic`, `extended`, `one_time`, любые);
- сортировки (по дате публикации новые сверху по умолчанию; альтернатива — по названию).

### 3.2. Состав карточки модуля
- обложка модуля (`coverImage`);
- название модуля;
- краткое описание (`summary`);
- индикатор уровня платности (`basic`/`extended`/`one_time`);
- цена для модулей `one_time`;
- число глав и уроков;
- прогресс пользователя — отображается только если у пользователя есть доступ к содержимому модуля и в модуле есть хотя бы один опубликованный урок (см. [progress.md](./progress.md), раздел 5.2). Формат: «X из N» и процент;
- индикатор доступа: «Доступен» / «Требуется повышение подписки» / «Требуется покупка» — зависит от состояния пользователя относительно `accessLevel` модуля.

### 3.3. Действие по карточке
При клике на карточку:
- если у пользователя есть доступ к содержимому модуля — открывается страница обучающего модуля (раздел 5);
- если доступа нет — открывается страница превью модуля (раздел 4).

Различение этих двух исходов выполняется на основе условий, описанных в `overview.md`, раздел 7.2.

### 3.4. Поведение каталога в состояниях компании
- `active`, `past_due` — все опубликованные модули отображаются согласно правилам раздела 3.2; индикатор доступа учитывает действующий тариф пользователя;
- `demo` — все опубликованные модули отображаются; модули `extended` и `one_time` показывают индикатор «Требуется повышение подписки» или «Требуется покупка»;
- `suspended`, `archived`, `blocked` — каталог недоступен.

## 4. Страница превью модуля

### 4.1. Когда отображается
Страница превью открывается, когда пользователь нажимает на карточку модуля в каталоге, к которому у него нет доступа (см. `overview.md`, раздел 7.2).

### 4.2. Состав экрана
- хлебные крошки: «Главная / Обучение / {название модуля}»;
- заголовок «{название модуля}»;
- обложка модуля (`coverImage`);
- продающее описание (`promotionalDescription`) — отрисовка markdown с заголовками, абзацами, изображениями, инлайн-ссылками;
- блок «Что вы узнаете в этом модуле» — список пунктов из `whatYouWillLearn`, отображается, если список не пустой;
- блок «Что внутри» — список глав модуля. Для каждой главы — её название и список уроков с названиями, без содержимого. Уроки в `draft` не отображаются;
- блок цены и доступа:
  - для модуля `extended` при базовой подписке пользователя — пояснение «Этот модуль входит в расширенную подписку» и кнопка «Повысить подписку» (переход в раздел биллинга `07-platform/subscriptions-and-billing.md`);
  - для модуля `one_time` — пояснение «Доступ к модулю — разовая покупка» с указанием цены и кнопка «Купить за N ₽» (переход в сценарий разовой покупки `07-platform/subscriptions-and-billing.md`, раздел 6);
  - для прочих ситуаций недоступности (например, `past_due` без активного тарифного покрытия) — пояснение и переход в раздел биллинга.

### 4.3. Поведение в состояниях компании
- `active`, `past_due` — открывается, если у пользователя нет доступа к содержимому;
- `demo` — открывается для модулей `extended` и `one_time`; для `basic` пользователь сразу попадает на страницу обучающего модуля (раздел 5);
- `suspended`, `archived`, `blocked` — недоступна.

### 4.4. Прогресс
На странице превью прогресс не отображается (см. [progress.md](./progress.md), раздел 5.3).

## 5. Страница обучающего модуля

### 5.1. Когда отображается
Страница обучающего модуля открывается, когда пользователь нажимает на карточку модуля в каталоге и у него есть доступ к содержимому модуля (`overview.md`, раздел 7.2).

### 5.2. Состав экрана
- хлебные крошки: «Главная / Обучение / {название модуля}»;
- заголовок «{название модуля}»;
- обложка модуля;
- индикатор уровня платности и индикатор «Доступно»;
- прогресс по модулю (`progress.md`, раздел 5.2): доля «X из N» и процентное значение;
- кнопка «Перейти к обучению» — переход к первому опубликованному уроку первой главы, либо к следующему непросмотренному уроку, если у пользователя уже есть прогресс. Кнопка не отображается, если в модуле нет опубликованных уроков (`progress.md`, раздел 5.2);
- полное описание модуля (`description`) — отрисовка markdown;
- оглавление с главами и уроками:
  - каждая глава — название и список её опубликованных уроков;
  - каждый урок — название, индикатор «Пройден» / «Не пройден»;
  - клик по уроку открывает страницу урока (раздел 6);
  - уроки в `draft` не отображаются;
  - главы без опубликованных уроков отображаются как пустые с пометкой «Уроки в подготовке».

### 5.3. Поведение в состояниях компании
- `active`, `past_due` — доступна при наличии тарифного покрытия;
- `demo` — доступна для модулей `basic`;
- `suspended`, `archived`, `blocked` — недоступна.

### 5.4. Свободная навигация
Пользователь может открывать любой опубликованный урок модуля в любом порядке. Линейная блокировка («следующий урок открывается только после прохождения предыдущего») в первой версии не применяется (см. `overview.md`, раздел 8).

## 6. Страница урока

### 6.1. Состав экрана
Страница урока состоит из:
- хлебных крошек: «Главная / Обучение / {название модуля} / {название главы} / {название урока}»;
- заголовка «{название урока}»;
- основного блока контента (раздел 6.2);
- боковой панели справа (раздел 6.3);
- кнопки «Отметить как пройденный» / индикатора «Урок пройден» под основным блоком (раздел 6.4);
- кнопок навигации между уроками (раздел 6.5).

### 6.2. Основной блок контента
В основном блоке последовательно отображаются блоки контента урока (`LessonContentBlock`) в порядке, заданном `position`. Отрисовка блоков по типам:

- `heading` — отображается как заголовок второго уровня;
- `subheading` — отображается как заголовок третьего уровня;
- `paragraph` — отрисовка markdown: жирный, курсив, инлайн-ссылки, маркированные и нумерованные списки;
- `image` — изображение по `fileId` с подписью `caption` под ним и `altText` как атрибут доступности;
- `gallery` — сетка миниатюр; клик по миниатюре открывает изображение в полноэкранном просмотре (lightbox) с возможностью переключения между изображениями галереи;
- `video` — встроенный плеер Rutube с использованием `rutubeUrl`; под плеером отображается опциональная подпись `caption`. Поведение при ошибках встраивания — раздел 7.

Между блоками сохраняется типовой вертикальный отступ. Дополнительное оформление (фоны, рамки, отдельные стили заголовков внутри блока) в первой версии не применяется.

### 6.3. Боковая панель
Боковая панель справа от основного блока содержит:
- блок «Прогресс курса» — название модуля, доля «X из N» уроков пройдено, процент;
- блок «Материалы урока» — список прикреплённых файлов (`LessonAttachment`) в порядке, заданном `position`. Каждый материал отображает иконку типа файла, отображаемое имя (`displayName`), размер и кнопку «Скачать». Клик скачивает файл по подписанной ссылке (`07-platform/storage.md`, раздел 5.11);
- блок «Оглавление модуля» (опционально, в зависимости от ширины экрана) — компактный список глав и уроков с подсветкой текущего урока.

На мобильных экранах боковая панель отображается под основным блоком как раскрывающиеся секции.

### 6.4. Отметка прогресса
Под основным блоком контента отображается кнопка «Отметить как пройденный». Поведение кнопки описано в [progress.md](./progress.md), раздел 3.

После нажатия кнопка превращается в неактивный индикатор «Урок пройден» с датой прохождения. Состояние сохраняется при перезагрузке страницы.

### 6.5. Навигация между уроками
В нижней части страницы урока — кнопки:
- «← Предыдущий урок» — открывает предыдущий опубликованный урок в текущей главе или последний опубликованный урок предыдущей главы. Кнопка не отображается, если текущий урок — самый первый в модуле;
- «Следующий урок →» — открывает следующий опубликованный урок в текущей главе или первый опубликованный урок следующей главы. Кнопка не отображается, если текущий урок — самый последний в модуле.

Кнопка «Следующий урок» доступна независимо от того, отметил ли пользователь текущий урок как пройденный.

### 6.6. Поведение в состояниях компании
- `active`, `past_due` — доступна при наличии тарифного покрытия модуля;
- `demo` — доступна для модулей `basic`;
- `suspended`, `archived`, `blocked` — недоступна.

## 7. Поведение при ошибках видео

### 7.1. Ролик Rutube недоступен
Если конкретный ролик Rutube возвращает ошибку (удалён, закрыт автором, ограничен по геолокации), на месте блока `video` отображается плашка «Видео временно недоступно» с указанием подписи `caption`, если она была задана. Остальные блоки урока, прикреплённые материалы и навигация остаются доступны.

### 7.2. Rutube как сервис недоступен
При массовой недоступности Rutube все блоки `video` всех уроков отображают плашку «Видео временно недоступно». Остальной контент уроков остаётся доступен.

### 7.3. Поведение прогресса при недоступном видео
Доступность видео не влияет на возможность отметить урок как пройденный. Кнопка «Отметить как пройденный» работает независимо от того, удалось ли пользователю просмотреть видео.

## 8. Поведение при отсутствии доступа в процессе чтения

### 8.1. Истечение подписки в процессе урока
Если в процессе просмотра урока подписка пользователя истекла (например, переход компании в `suspended`), на следующее обновление страницы пользователь перенаправляется на страницу превью модуля или, при полной потере доступа, на главную страницу кабинета с уведомлением об ограничении доступа.

В рамках текущей сессии до обновления страницы возможно дочитать урок без перезагрузки. Платформа не выполняет агрессивный middleware-перехват в активной сессии.

### 8.2. Возврат активного доступа
При возврате компании в `active` пользователь снова видит каталог, страницы модулей и страницы уроков. Прогресс сохраняется (см. [progress.md](./progress.md), раздел 6).

## 9. Открытые вопросы первой версии

### 9.1. Поведение при медленной загрузке embed Rutube
Параметры таймаута на загрузку embed-фрейма Rutube и поведение при долгой загрузке (показ спиннера, плашки об ошибке после N секунд) — открытый вопрос. Конкретное решение фиксируется в техническом задании на интеграцию с Rutube.

### 9.2. Адаптивность mobile
Подробная адаптация боковой панели и оглавления под мобильные экраны (точные брейкпоинты, поведение раскрывающихся секций) — на стороне дизайн-системы.

### 9.3. Звук видео по умолчанию
Поведение плеера Rutube по умолчанию (автозапуск, mute) — определяется самим Rutube и его настройками встраивания. Платформа не управляет этими параметрами в первой версии.
