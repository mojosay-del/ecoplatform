## 1. Назначение раздела
Настоящий документ фиксирует состав сущностей модуля «Обучение», их атрибуты, связи между ними, правила валидации и поведение. Документ продолжает [overview.md](./overview.md) и развивает разделы 5 и 6 шапки.

Документ определяет:
- состав полей сущностей `LearningModule`, `LearningModulePreview`, `Chapter`, `Lesson`, `LessonContentBlock`, `LessonAttachment`;
- типы блоков контента урока и состав их полей;
- правила добавления, редактирования и удаления элементов иерархии;
- правила валидации перед публикацией;
- поведение при ошибках встраивания видео;
- каскадные эффекты при удалении.

Состав поля для модели данных подлежит окончательной фиксации в `08-architecture/data-model.md`. Настоящий документ задаёт продуктовый набор полей.

## 2. LearningModule — обучающий модуль

### 2.1. Назначение
Корневая единица обучения. Объединяет главы и уроки по одной теме. Имеет уровень платности, состояние публикации, обложку, описания и страницу превью (раздел 3).

### 2.2. Состав полей
- `id` — идентификатор;
- `title` — название модуля (строка, обязательное, лимиты — `limits.md`, раздел 2);
- `summary` — краткое описание для карточки в каталоге (строка, обязательное);
- `description` — полное описание модуля для страницы модуля (markdown, обязательное);
- `coverImage` — обложка модуля (ссылка на файл в файловом хранилище, обязательное при первой публикации);
- `accessLevel` — уровень платности: `basic`, `extended`, `one_time` (обязательное; см. `overview.md`, раздел 7.1);
- `oneTimePrice` — цена разовой покупки в рублях (число, обязательное при `accessLevel = one_time`, иначе не используется);
- `status` — состояние публикации: `draft`, `published`;
- `firstPublishedAt` — момент первой публикации (заполняется автоматически);
- `lastUpdatedAt` — момент последней редакции (заполняется автоматически);
- `createdBy` — идентификатор контент-менеджера, создавшего модуль;
- `chapters` — упорядоченный список глав (раздел 4);
- `preview` — связь с превью модуля (раздел 3); один к одному.

### 2.3. Состояния публикации
- `draft` — модуль виден только в админ-панели; в каталоге пользователей не отображается;
- `published` — модуль виден в каталоге пользователям, у которых уровень подписки или разовая покупка покрывают `accessLevel`.

Переходы: `draft → published` (контент-менеджер или администратор), `published → draft` (только администратор).

Полные правила переходов и валидация перед публикацией — [cabinet-content-manager.md](./cabinet-content-manager.md).

## 3. LearningModulePreview — превью модуля

### 3.1. Назначение
Продающая страница модуля. Отображается пользователю, у которого нет доступа к содержимому (см. `overview.md`, раздел 7.3).

### 3.2. Состав полей
- `id` — идентификатор;
- `moduleId` — ссылка на обучающий модуль (один к одному);
- `promotionalDescription` — продающее описание (markdown с заголовками, абзацами, изображениями, инлайн-ссылками; обязательное при первой публикации модуля);
- `whatYouWillLearn` — список пунктов «Что вы узнаете в этом модуле» (упорядоченный список строк, опциональный);
- `lastUpdatedAt` — момент последней редакции.

Дополнительные элементы превью (название, обложка, оглавление, цена) автоматически берутся из соответствующих полей `LearningModule`, не дублируются в превью.

### 3.3. Связь с модулем
Превью создаётся вместе с модулем как пустая запись (`promotionalDescription` пустой, `whatYouWillLearn` пустой). Заполняется контент-менеджером перед первой публикацией.

Перед первой публикацией модуля поле `promotionalDescription` обязательно к заполнению (раздел 9.2). После публикации редактируется свободно.

## 4. Chapter — глава

### 4.1. Назначение
Промежуточный группирующий элемент внутри обучающего модуля. Содержит уроки по одной подтеме модуля.

### 4.2. Состав полей
- `id` — идентификатор;
- `moduleId` — ссылка на обучающий модуль;
- `title` — название главы (строка, обязательное);
- `position` — порядковый номер главы внутри модуля (целое число; уникально в пределах модуля);
- `lessons` — упорядоченный список уроков (раздел 5).

Глава не имеет собственного описания, обложки и состояния публикации. Глава всегда видна, если виден её модуль и хотя бы один её урок.

## 5. Lesson — урок

### 5.1. Назначение
Листовая единица обучения. Страница из произвольной последовательности блоков контента (раздел 6).

### 5.2. Состав полей
- `id` — идентификатор;
- `chapterId` — ссылка на главу;
- `title` — название урока (строка, обязательное);
- `position` — порядковый номер урока внутри главы (целое число; уникально в пределах главы);
- `status` — состояние публикации: `draft`, `published`;
- `firstPublishedAt` — момент первой публикации;
- `lastUpdatedAt` — момент последней редакции;
- `contentBlocks` — упорядоченный список блоков контента (раздел 6);
- `attachments` — список прикреплённых материалов (раздел 7).

### 5.3. Состояния публикации
- `draft` — урок виден только в админ-панели; в публичном интерфейсе модуля не отображается;
- `published` — урок виден пользователю при условии, что модуль также `published` и у пользователя есть доступ к содержимому модуля (см. `overview.md`, раздел 7).

Переходы: `draft → published` (контент-менеджер или администратор), `published → draft` (только администратор).

Если модуль в состоянии `draft`, ни один урок этого модуля не виден пользователю независимо от состояния урока.

Если модуль в состоянии `published`, но конкретный урок в состоянии `draft`, в оглавлении модуля этот урок не отображается. Опубликованные уроки сохраняют свою нумерацию и порядок без учёта черновиков.

## 6. LessonContentBlock — блок контента урока

### 6.1. Назначение
Упорядоченный элемент тела урока. Урок состоит из последовательности блоков; ни один тип блока не является обязательным; порядок задаётся контент-менеджером.

### 6.2. Состав полей
- `id` — идентификатор;
- `lessonId` — ссылка на урок;
- `position` — порядковый номер блока внутри урока (целое число; уникально в пределах урока);
- `type` — тип блока (раздел 6.3);
- `payload` — структура полей, специфичная для типа блока (раздел 6.3).

### 6.3. Типы блоков
В первой версии поддерживаются шесть типов блоков:

#### 6.3.1. `heading` — заголовок
Заголовок секции урока. Используется для крупных разделительных заголовков внутри урока.

Поля `payload`:
- `text` — текст заголовка (строка, обязательное; лимиты — `limits.md`, раздел 5).

#### 6.3.2. `subheading` — подзаголовок
Подзаголовок внутри секции. Используется для подразделов в пределах одного `heading`.

Поля `payload`:
- `text` — текст подзаголовка (строка, обязательное; лимиты — `limits.md`, раздел 5).

#### 6.3.3. `paragraph` — абзац
Текстовый абзац в формате markdown. Поддерживает инлайн-форматирование (жирный, курсив), инлайн-ссылки и списки. Не поддерживает заголовки внутри абзаца (для них есть отдельные блоки `heading` и `subheading`) и встраивание изображений или видео внутри markdown (для них есть отдельные блоки).

Поля `payload`:
- `markdown` — текст абзаца (строка, обязательное; лимиты — `limits.md`, раздел 5).

#### 6.3.4. `image` — изображение
Одиночное изображение с опциональной подписью.

Поля `payload`:
- `fileId` — ссылка на файл изображения в файловом хранилище (обязательное; уровень доступа `public`; см. `07-platform/storage.md`, раздел 5.11);
- `caption` — подпись под изображением (строка, опциональная);
- `altText` — альтернативный текст для доступности (строка, опциональная).

#### 6.3.5. `gallery` — галерея изображений
Упорядоченный набор изображений. Отображается в публичном интерфейсе как сетка миниатюр с возможностью просмотра в полном размере (lightbox).

Поля `payload`:
- `images` — упорядоченный список изображений (раздел 6.3.4: `fileId`, `caption`, `altText`); список не может быть пустым.

#### 6.3.6. `video` — видео
Встроенный видеоролик с внешнего хостинга Rutube.

Поля `payload`:
- `rutubeUrl` — URL ролика Rutube (строка, обязательное; должна быть валидной ссылкой на ролик Rutube — раздел 8);
- `caption` — подпись под видео (строка, опциональная).

### 6.4. Поведение при удалении файла блока
Если файл изображения или галереи (поле `fileId`) удалён из файлового хранилища (например, вручную администратором через сценарий поддержки), при отображении урока на месте блока выводится плашка «Изображение недоступно» с указанием идентификатора блока. Контент-менеджер видит ту же плашку в редакторе и заменяет файл новым.

## 7. LessonAttachment — прикреплённый материал

### 7.1. Назначение
Скачиваемый файл, привязанный к уроку. Отображается в боковой панели страницы урока (см. `overview.md`, раздел 10).

### 7.2. Состав полей
- `id` — идентификатор;
- `lessonId` — ссылка на урок;
- `fileId` — ссылка на файл в файловом хранилище (обязательное; уровень доступа `public` либо `authenticated` — определяется правилами модуля и фиксируется в `07-platform/storage.md`, раздел 5.11);
- `displayName` — отображаемое имя материала на странице урока (строка, обязательное); по умолчанию равно имени файла, контент-менеджер может переопределить;
- `position` — порядковый номер материала в боковой панели (целое число; уникально в пределах урока).

Допустимые типы файлов и лимиты — `limits.md`, раздел 6.

## 8. Видео Rutube — встраивание

### 8.1. Формат URL
В поле `rutubeUrl` блока `video` (раздел 6.3.6) принимаются ссылки на отдельные ролики Rutube. Платформа выполняет минимальную валидацию: домен ссылки должен принадлежать Rutube, путь должен соответствовать публичному ролику.

Ссылки на каналы, плейлисты, шорты, прямые трансляции в первой версии не поддерживаются.

### 8.2. Сценарий встраивания
При сохранении блока `video` платформа сохраняет введённый URL без обращения к Rutube. На странице урока (см. `public-ui.md`) URL подставляется в стандартный embed-код Rutube для отображения плеера.

Платформа не загружает видео, не получает метаданные ролика (название, длительность, превью), не сохраняет копию.

### 8.3. Поведение при ошибках встраивания
- **Ролик удалён, закрыт автором, недоступен по геолокации:** на месте блока в публичном интерфейсе отображается плашка «Видео временно недоступно». Остальные блоки и материалы урока остаются доступны.
- **Ссылка некорректна (ошибка валидации формата):** при сохранении блока контент-менеджер получает сообщение об ошибке; блок не сохраняется до исправления.
- **Rutube как сервис недоступен (массовая ошибка):** на месте блоков `video` в публичном интерфейсе отображается плашка «Видео временно недоступно». Остальной контент урока остаётся доступен.

Оповещение администратора платформы о массовой недоступности Rutube в первой версии не реализуется (открытый вопрос для последующих версий).

## 9. Правила валидации перед публикацией

### 9.1. Валидация урока перед `draft → published`
Урок может быть переведён в `published`, если выполнены условия:
- `title` заполнен;
- урок содержит хотя бы один блок контента (любого типа);
- все блоки прошли валидацию своих полей (раздел 6.3);
- все ссылки на файлы в блоках указывают на существующие файлы в файловом хранилище.

### 9.2. Валидация модуля перед первой публикацией
Модуль может быть переведён в `published` впервые, если выполнены условия:
- `title`, `summary`, `description` заполнены;
- `coverImage` указывает на существующий файл;
- `accessLevel` задан; для `accessLevel = one_time` задан `oneTimePrice` больше нуля;
- модуль содержит хотя бы одну главу;
- хотя бы одна глава содержит хотя бы один опубликованный урок;
- превью модуля заполнено: `promotionalDescription` не пустое.

### 9.3. Повторная публикация модуля
При снятии модуля с публикации (`published → draft`, только администратор) и повторной публикации проверки раздела 9.2 выполняются заново. История прошлых публикаций не используется для упрощения проверок.

## 10. Удаление и каскады

### 10.1. Удаление урока
- **Контент-менеджер** удаляет урок только в состоянии `draft`. Удаление урока в `published` запрещено.
- **Администратор** удаляет урок в любом состоянии.

При удалении урока удаляются:
- все блоки контента этого урока;
- все записи `LessonAttachment` этого урока (сами файлы в хранилище — по правилам `07-platform/storage.md`);
- все записи `LessonProgress` пользователей по этому уроку (см. [progress.md](./progress.md)).

### 10.2. Удаление главы
- **Контент-менеджер** удаляет главу только в состоянии модуля `draft`. Удаление главы в опубликованном модуле запрещено.
- **Администратор** удаляет главу в любом состоянии модуля.

При удалении главы каскадно удаляются все её уроки со всеми последствиями раздела 10.1.

Глава не может быть удалена, если она единственная в опубликованном модуле и без неё модуль перестаёт удовлетворять требованию «модуль содержит хотя бы одну главу» (раздел 6 overview.md). Действие администратора в этом случае — сначала перевести модуль в `draft`, затем удалить главу.

### 10.3. Удаление модуля
- **Контент-менеджер** удаляет модуль только в состоянии `draft`, не имеющий ранее опубликованных редакций (`firstPublishedAt` не заполнен).
- **Администратор** удаляет модуль в любом состоянии.

При удалении модуля каскадно удаляются:
- превью модуля;
- все главы модуля;
- все уроки этих глав;
- все блоки контента этих уроков;
- все материалы этих уроков;
- все записи `LessonProgress` пользователей по этим урокам.

Удаление одобренных разовых покупок (`OneTimePurchase`), относящихся к удаляемому модулю `one_time`, не выполняется. Записи покупок сохраняются для целей биллинговой истории и налоговой отчётности (`07-platform/subscriptions-and-billing.md`, раздел 6.5). Финансовые документы по этим покупкам также сохраняются.

### 10.4. Журналирование
Все удаления модулей, глав и уроков фиксируются в `AdminActionLog` (`07-platform/admin-panel.md`, раздел 16) с указанием актора, времени, типа сущности, идентификатора и причины. Причина обязательна при удалении опубликованных сущностей администратором; для удаления черновиков контент-менеджером причина опциональна.
