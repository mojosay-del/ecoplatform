## 1. Назначение раздела
Настоящий документ фиксирует устройство публичного интерфейса модуля «Индексы цен» — экранов, которые видит пользовательский аккаунт компании в кабинете платформы.

Документ продолжает [overview.md](./overview.md), раздел 10, и развивает каждый элемент публичного экрана до уровня состава блоков и поведения.

Документ определяет:
- точку входа в модуль из бокового меню кабинета;
- раздел «Индексы цен» с вкладками групп и сеткой карточек;
- состав карточки индекса;
- работу графика и тултипа;
- работу селектора периода;
- поведение в состояниях компании;
- поведение при отсутствии данных.

## 2. Точка входа
В боковом меню кабинета пользовательского аккаунта компании присутствует пункт «Индексы цен» в разделе «Главная». При нажатии открывается экран «Индексы цен» (раздел 3).

Точка входа отображается всем типам компаний (заготовитель, трейдер, переработчик) с одинаковым составом модуля.

## 3. Раздел «Индексы цен»

### 3.1. Состав экрана
Экран состоит из:
- заголовка раздела «Индексы цен на вторсырьё»;
- подзаголовка-пояснения (например, «Актуальные ценовые индексы по основным категориям сырья»);
- горизонтальной панели вкладок групп (раздел 3.2);
- сетки карточек индексов в активной группе (раздел 4).

### 3.2. Вкладки групп
Над сеткой карточек отображается горизонтальная панель вкладок:
- одна вкладка на каждую активную категорию `NomenclatureCategory` (`isActive = true`);
- порядок вкладок задаётся полем `position` категории;
- активная вкладка визуально выделена (например, тёмным фоном);
- неактивные вкладки кликабельны, их выбор переключает сетку карточек.

При первой загрузке раздела активна первая по порядку вкладка (с наименьшим `position`).

При переключении вкладки сетка карточек пересчитывается под новую категорию без перезагрузки страницы. URL обновляется на `/indices/category/{slug}`.

При прямом переходе по URL `/indices/category/{slug}` соответствующая вкладка активируется при загрузке страницы. Если категории с указанным `slug` не существует или она деактивирована, открывается первая активная категория.

### 3.3. Поведение при отсутствии активных категорий
Если в системе нет ни одной активной категории (или ни одна категория не содержит опубликованных индексов), отображается пустой состояние: «Индексы цен ещё не опубликованы. Раздел будет дополнен скоро».

### 3.4. Поведение в состояниях компании
- `active`, `past_due`, `demo` — раздел доступен в полном объёме;
- `suspended`, `archived`, `blocked` — раздел недоступен.

## 4. Сетка карточек индексов

### 4.1. Состав сетки
В активной вкладке отображается сетка карточек индексов:
- одна карточка на каждый опубликованный индекс активной категории (раздел 5.4 [indices.md](./indices.md) — деактивированные номенклатуры скрыты);
- сортировка карточек — по названию номенклатуры в алфавитном порядке.

Альтернативные сортировки (по недельному изменению, по тренду) в первой версии не предусматриваются.

### 4.2. Адаптивность
- На широких экранах — три колонки;
- На средних — две колонки;
- На мобильных — одна колонка.

Конкретные брейкпоинты — на стороне дизайн-системы.

### 4.3. Поведение при отсутствии карточек
Если в активной категории нет ни одного опубликованного индекса, отображается пустое состояние: «В этой категории пока нет опубликованных индексов».

## 5. Карточка индекса

### 5.1. Состав карточки
Каждая карточка индекса содержит:
- **Название номенклатуры** (например, «Картон»);
- **Код номенклатуры** (например, «МКР-КРТ-001») мелким шрифтом под названием;
- **Текущая цена** в правом верхнем углу карточки в формате «14 250 ₽/т» (раздел 5.2);
- **Недельное изменение** в формате «+1.8% за неделю» (зелёный цвет для роста, красный — для падения, серый — для нулевого изменения; раздел 5.3);
- **Плашка тренда** — «Рост» / «Стагнация» / «Падение» (раздел 5.4);
- **Селектор периода** — четыре кнопки 1М / 3М / 6М / 1Г (раздел 6);
- **Мини-график** с подписями месяцев на оси X и тултипом при наведении (раздел 7).

Между элементами сохраняются типовые отступы дизайн-системы. Дополнительные поля карточки в первой версии не отображаются (служебное описание индекса, иллюстрация номенклатуры — нет).

### 5.2. Текущая цена
Текущая цена — `currentPrice` из расчёта раздела 5.1 [indices.md](./indices.md). Формат отображения:
- разделитель тысяч — пробел («14 250»);
- единица измерения номенклатуры — после числа («₽/т»).

Если у индекса нет ни одного значения с `date ≤ today` — индекс на странице не отображается ([overview.md](./overview.md), раздел 6).

### 5.3. Недельное изменение
Недельное изменение — `weeklyChange` из расчёта раздела 5.2 [indices.md](./indices.md). Формат отображения:
- знак — обязателен («+», «−», «±» для нуля);
- одна цифра после запятой («+1.8%», «−2.1%», «±0.0%»);
- цвет:
  - `weeklyChange > 0` — зелёный (рост);
  - `weeklyChange < 0` — красный (падение);
  - `weeklyChange = 0` — серый (без изменений).

Если недельное изменение не рассчитывается (раздел 5.4 [indices.md](./indices.md) — нет данных в окне 14 дней) — недельное изменение и плашка тренда не отображаются. Текущая цена остаётся видимой.

### 5.4. Плашка тренда
Плашка тренда отображается рядом с недельным изменением. Состояние плашки определяется по порогам (раздел 8.3 [overview.md](./overview.md)):
- зелёная плашка «Рост»;
- серая плашка «Стагнация»;
- красная плашка «Падение».

В первой версии плашка не содержит дополнительной информации помимо названия и цвета. Численное значение `weeklyChange` отображается отдельно (раздел 5.3).

## 6. Селектор периода

### 6.1. Состав
Над графиком на каждой карточке отображается панель из четырёх кнопок:
- `1М` — 30 дней;
- `3М` — 90 дней (выделена по умолчанию);
- `6М` — 180 дней;
- `1Г` — 365 дней.

Активная кнопка визуально выделена (тёмный фон, белый текст). Неактивные кнопки нейтрального цвета.

### 6.2. Период по умолчанию
По умолчанию активен период `3М`. При первом открытии раздела все карточки отображают график за последние 90 дней.

Сохранение выбора пользователя между сессиями в первой версии не предусматривается. При следующем открытии раздела все карточки снова на `3М`.

### 6.3. Переключение периода
Клик на кнопку периода:
- меняет активную кнопку;
- пересчитывает график карточки под выбранный период (раздел 7.4);
- никак не влияет на другие карточки и на другие элементы карточки (название, текущая цена, недельное изменение, плашка тренда не меняются).

Селектор периода работает независимо для каждой карточки. Пользователь может одновременно видеть карточку «Картон» с периодом `1М` и карточку «Бумага микс» с периодом `1Г`.

### 6.4. Поведение при недостаточных данных
Все четыре кнопки доступны для нажатия независимо от объёма исторических данных. Если данных меньше выбранного периода — график показывает все имеющиеся точки в пределах окна (раздел 7.2 [indices.md](./indices.md)).

## 7. Мини-график

### 7.1. Состав
Мини-график занимает нижнюю часть карточки (примерно две трети высоты). Содержит:
- горизонтальную ось X с подписями месяцев;
- ломаную линию, соединяющую точки исторических значений;
- градиентную заливку области под линией (декоративный элемент дизайн-системы);
- маркер последней точки (текущей цены) в виде цветной точки;
- вертикальную пунктирную линию от точки «Текущая цена» до соответствующей подписи месяца с подписью значения цены.

### 7.2. Подписи на оси X
Ось X разбита на месяцы. Под каждым месяцем — короткая подпись (например, «Мар», «Апр», «Май», «Июн», «Июл», «Авг»).

Число месяцев на оси соответствует выбранному периоду:
- `1М` — текущий и предыдущий месяцы;
- `3М` — текущий и три предыдущих;
- `6М` — текущий и пять-шесть предыдущих;
- `1Г` — все 12 месяцев года.

Конкретное оформление подписей и тиков — на стороне дизайн-системы.

### 7.3. Линия графика
Линия — ломаная, соединяющая существующие точки `PriceIndexValue` в порядке возрастания даты. Между датами без значений линия идёт прямо от точки к точке без интерполяции.

Если в выбранном периоде только одна точка — отображается одна точка без линии. Если ноль точек — график пустой с пометкой «Нет данных за выбранный период» (минимальная служебная плашка внутри области графика).

### 7.4. Пересчёт при смене периода
При клике на кнопку периода график пересчитывается:
- определяется окно `[today - {период в днях}, today]`;
- из всех значений индекса выбираются записи с `date` в этом окне;
- строится ломаная линия по выбранным точкам;
- пересчитываются подписи на оси X.

### 7.5. Тултип при наведении
При наведении курсора на любую часть графика отображается тултип:
- черная закруглённая плашка с белым текстом;
- содержание плашки — значение цены в формате «14 250» (без единицы измерения, она и так понятна по контексту);
- вертикальная пунктирная линия от тултипа к подписи месяца на оси X;
- если курсор находится не точно на точке `PriceIndexValue`, тултип показывает значение ближайшей по дате точки;
- при выводе курсора за пределы графика тултип исчезает; на месте маркера остаётся индикатор последней (текущей) точки.

### 7.6. Маркер последней точки
В состоянии «курсор не наведён на график» в графике отображается маркер последней (актуальной) точки — цветная точка на ломаной с тултипом, указывающим текущую цену. Это даёт визуальную привязку при первом взгляде на карточку.

### 7.7. Tap-поведение на мобильных
На мобильных устройствах вместо ховера используется тап:
- тап в любой точке графика — отображает тултип на ближайшей по дате точке;
- тап в стороне от графика — скрывает тултип;
- маркер последней точки сохраняется как до тапа, так и после скрытия тултипа.

## 8. Поведение при ошибках и пустых состояниях

### 8.1. Индекс без значений в окне
Если индекс имеет опубликованное состояние, но в выбранном периоде нет ни одной точки — на месте графика отображается плашка «Нет данных за выбранный период». Карточка остаётся с текущей ценой и недельным изменением (если они рассчитываются — раздел 5.3).

### 8.2. Деактивация номенклатуры в реальном времени
Если администратор деактивирует номенклатуру, для которой у пользователя открыта карточка индекса, при следующей загрузке страницы или при переключении вкладки эта карточка исчезает. В рамках текущей сессии до обновления страницы карточка остаётся видимой.

### 8.3. Удаление индекса в реальном времени
Аналогично — удалённый администратором индекс исчезает с карточки при следующем обращении пользователя к разделу. В текущей сессии до обновления страницы карточка остаётся видимой.

### 8.4. Потеря доступа в процессе чтения
Если в процессе просмотра подписка пользователя истекла и компания перешла в `suspended` или иное состояние с закрытым доступом, на следующее обновление страницы пользователь перенаправляется на главную страницу кабинета с уведомлением об ограничении доступа.

В рамках текущей сессии до обновления страницы пользователь продолжает видеть раздел.

## 9. Открытые вопросы первой версии

### 9.1. Сравнение нескольких индексов на одном графике
Возможность пользователю выбрать несколько индексов и наложить их кривые на один график для сравнения в первой версии не реализуется ([overview.md](./overview.md), раздел 4).

### 9.2. Экспорт данных пользователем
Кнопка «Экспортировать в CSV» на карточке индекса в первой версии не предусматривается. Пользователь работает только с визуальным представлением.

### 9.3. Сортировки и фильтры
Альтернативные сортировки карточек (по недельному изменению, по тренду) и фильтры (только растущие, только падающие) в первой версии не предусматриваются (раздел 4.1).

### 9.4. Сохранение выбранного периода между сессиями
В первой версии выбор периода не сохраняется между сессиями (раздел 6.2). Все карточки при следующем открытии снова на `3М`.

### 9.5. Анимация графика
Параметры анимации перерисовки графика при смене периода (плавный переход или мгновенная замена) — на стороне дизайн-системы. Семантически оба варианта приемлемы.

### 9.6. Иллюстрации видов сырья
Использование иллюстраций номенклатуры (`07-platform/storage.md`, раздел 5.4) на карточке индекса в первой версии не реализуется. Карточка содержит только текстовые данные и график (раздел 5.1, [overview.md](./overview.md), раздел 11.3).
