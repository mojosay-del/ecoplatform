## 1. Назначение раздела
Настоящий документ фиксирует точки интеграции модуля «Индексы цен» со смежными подсистемами платформы: подписками и биллингом, справочником номенклатуры, файловым хранилищем, уведомлениями, модерацией, административной панелью и торговой площадкой.

Документ продолжает [overview.md](./overview.md), раздел 11, и развивает каждую интеграцию до уровня конкретных правил поведения.

Документ определяет:
- влияние подписки на доступ к разделу «Индексы цен»;
- сопутствующую правку к `07-platform/admin-panel.md`, раздел 13.2 — продвижение поля «категория» номенклатуры в первоклассную сущность `NomenclatureCategory`;
- отсутствие специфичных уведомлений модуля в первой версии;
- неприменение модерации к индексам;
- размещение раздела в админ-панели;
- связь с торговой площадкой.

## 2. Подписки и биллинг

### 2.1. Доступ к индексам
Все опубликованные индексы доступны при наличии любой действующей подписки платформы (см. [overview.md](./overview.md), раздел 7.1). Гейтинга на уровне отдельных индексов или категорий нет.

Это согласовано с моделью подписок (`07-platform/subscriptions-and-billing.md`, раздел 4) и мастер-документом продукта (`00-product/master.md`, раздел 9.2).

### 2.2. Состояния компании
Поведение доступа в состояниях компании, отличных от `active`:

| Состояние компании | Раздел «Индексы цен» |
|---|---|
| `active` | доступен |
| `demo` | доступен |
| `past_due` | доступен |
| `suspended` | недоступен |
| `archived` | недоступен |
| `blocked` | недоступен |

### 2.3. Биллинговые события
Модуль «Индексы цен» не порождает событий биллинга. Покупки внутри модуля (например, разовая покупка доступа к одному индексу) в первой версии не предусматриваются.

## 3. Файловое хранилище
Модуль «Индексы цен» в первой версии не использует файловое хранилище. У индекса нет прикреплённых файлов, изображений и медиа. Карточка индекса содержит только текстовые данные и график, отрисованный из значений базы данных.

Сопутствующие правки к `07-platform/storage.md` для модуля не требуются.

Использование иллюстраций номенклатуры (`07-platform/storage.md`, раздел 5.4) на карточке индекса — открытый вопрос для последующих версий ([overview.md](./overview.md), раздел 11.3; [public-ui.md](./public-ui.md), раздел 9.6).

## 4. Справочник номенклатуры — сопутствующая правка к `admin-panel.md`

### 4.1. Текущая модель в `admin-panel.md`
В `07-platform/admin-panel.md`, раздел 13.2 текущая модель справочника номенклатуры включает поле «категория» как простой текстовый атрибут номенклатуры:

> Состав записи справочника номенклатуры:
> - код вида сырья;
> - название;
> - категория (при наличии);
> - единица измерения;
> - описание;
> - признак активности.

### 4.2. Цель правки
Модуль «Индексы цен» использует категории номенклатуры как вкладки в публичной части (см. [public-ui.md](./public-ui.md), раздел 3.2). Для управления порядком вкладок, переименованием категорий, склейкой и удалением требуется первоклассная сущность.

Сопутствующая правка — продвижение «категории» в первоклассную сущность `NomenclatureCategory` (см. [indices.md](./indices.md), раздел 2).

### 4.3. Содержание правки
**Изменения в `07-platform/admin-panel.md`, раздел 13.2:**

1. **Состав записи справочника номенклатуры** — поле «категория (при наличии)» заменяется на поле `categoryId` (ссылка на `NomenclatureCategory`):

> Состав записи справочника номенклатуры:
> - код вида сырья;
> - название;
> - категория (`categoryId`, ссылка на `NomenclatureCategory`);
> - единица измерения;
> - описание;
> - признак активности.

2. **Добавление подраздела «Категории номенклатуры»** — описание сущности `NomenclatureCategory` со следующим составом полей:

> Состав записи справочника категорий номенклатуры:
> - идентификатор;
> - название;
> - слаг для URL вкладки в публичной части модулей;
> - порядок отображения;
> - признак активности.

3. **Описание управления категориями администратором** — добавление, переименование, изменение порядка, деактивация, удаление. Подробности — [cabinet-content-manager.md](./cabinet-content-manager.md), раздел 4 настоящего модуля.

4. **Расширение `AdminActionLog`** — действия с категориями номенклатуры фиксируются в журнале (создание, переименование, изменение порядка, деактивация, удаление).

### 4.4. Миграция существующих данных
В момент применения сопутствующей правки в системе уже могут существовать номенклатуры с заполненным текстовым полем «категория». Миграция:

1. Извлекаются все уникальные значения текстового поля «категория» по всем номенклатурам;
2. По каждому уникальному значению создаётся запись `NomenclatureCategory` с соответствующим `name`. Поле `position` назначается в порядке частоты встречаемости (наиболее частые — первыми), `slug` генерируется автоматически, `isActive = true`;
3. Для каждой номенклатуры её текстовое поле «категория» заменяется на `categoryId`, ссылающийся на соответствующую запись `NomenclatureCategory`;
4. Номенклатуры без заполненной категории получают `categoryId = null` (но в первой версии модуля «Индексы цен» (см. [indices.md](./indices.md), раздел 2.4) категория обязательна; администратор обязан перевести такие номенклатуры в одну из категорий вручную);
5. Текстовое поле «категория» удаляется из схемы.

Конкретный план миграции — `08-architecture/data-model.md` при применении сопутствующей правки.

### 4.5. Совместимость с торговой площадкой
Модуль «Торговая площадка» (`04-marketplace/`) использует тот же платформенный справочник номенклатуры. После применения сопутствующей правки торговая площадка получит ту же первоклассную сущность `NomenclatureCategory`. Использовать категории в торговой площадке (например, для группировки видов сырья в форме создания объявления) — открытый вопрос для последующих версий.

В первой версии торговая площадка не группирует виды сырья по категориям в UI — пользователь видит плоский список номенклатур (см. `04-marketplace/listings.md`, раздел 8.1: «вид сырья из справочника номенклатуры»). Изменение этого поведения не входит в настоящую правку.

## 5. Уведомления

### 5.1. Отсутствие специфичных событий в первой версии
Подсистема уведомлений (`07-platform/notifications.md`) в первой версии не порождает событий, специфичных для модуля «Индексы цен». Каталог событий не расширяется.

В частности, в первой версии не реализованы:
- уведомление о публикации нового индекса;
- уведомление о значимом изменении цены (например, рост на N% за неделю);
- уведомление пользователя о появлении новой категории;
- уведомление контент-менеджера об устаревших индексах (без обновлений за N дней).

### 5.2. Системные рассылки
Администратор платформы может информировать пользователей о публикации новых индексов через системные рассылки (`07-platform/admin-panel.md`, раздел 17). Это не специфичный для индексов механизм, а общая функция админ-панели.

### 5.3. Расширение каталога — открытый вопрос
Добавление специфичных уведомлений модуля «Индексы цен» в каталог событий — открытый вопрос для последующих версий. Особенно ценны:
- уведомления пользователю о значимых изменениях цен по индексам, на которые он подписан (требует подсистемы подписок на индексы);
- уведомления контент-менеджеру об индексах без обновлений (для контроля актуальности данных).

## 6. Модерация

### 6.1. Не применяется
Контент модуля «Индексы цен» создаётся платформенным сотрудником (контент-менеджером), не пользователями компаний. Поэтому модерация (`07-platform/moderation.md`) к индексам не применяется.

Снятие индекса с публикации администратором — административное действие в рамках workflow раздела (см. [cabinet-content-manager.md](./cabinet-content-manager.md), раздел 7.3), не модерационная санкция. Запись в журнал кейсов модерации не создаётся, в `AdminActionLog` действие фиксируется как административное.

### 6.2. Жалобы пользователей на индексы
В первой версии не предусмотрена форма подачи жалобы на индекс (например, «цена в индексе не соответствует рынку» или «значение явно ошибочное»). Обратная связь от пользователей собирается через обращения в поддержку (`07-platform/admin-panel.md`, раздел 14) с категорией `other`.

Подключение специализированного механизма обратной связи по индексам — открытый вопрос для последующих версий.

## 7. Административная панель

### 7.1. Размещение раздела
Раздел «Индексы цен» в админ-панели — часть структуры `07-platform/admin-panel.md`. Доступен платформенным сотрудникам с ролями `content_manager` и `admin`. Подробное устройство — [cabinet-content-manager.md](./cabinet-content-manager.md).

### 7.2. Журналирование
Действия контент-менеджера и администратора в части индексов цен частично фиксируются в едином `AdminActionLog` (`07-platform/admin-panel.md`, раздел 16). Перечень фиксируемых событий — [cabinet-content-manager.md](./cabinet-content-manager.md), раздел 10.

### 7.3. Аналитика
Аналитика использования индексов (число просмотров карточек, популярные категории, частота переключения периодов) в первой версии не реализуется (`07-platform/admin-panel.md`, раздел 19.5).

### 7.4. Служебные настройки
Параметр `stagnationThreshold` (порог стагнации для расчёта плашки тренда) хранится в служебных настройках платформы (`07-platform/admin-panel.md`, раздел 18). Изменение требует прав администратора.

## 8. Связь с торговой площадкой

### 8.1. Общий справочник номенклатуры
Модуль «Торговая площадка» (`04-marketplace/`) и модуль «Индексы цен» используют один и тот же платформенный справочник номенклатуры. Изменение справочника (добавление, переименование, деактивация номенклатуры) администратором отражается одновременно в обоих модулях.

### 8.2. Деактивация номенклатуры
При деактивации номенклатуры (`isActive = false`):
- индекс по этой номенклатуре сохраняет свои исторические значения, но скрывается в публичной части (раздел 3.4 [indices.md](./indices.md));
- номенклатура исчезает из доступных для выбора при создании объявления на торговой площадке (`04-marketplace/listings.md`, раздел 8.1);
- существующие объявления с этой номенклатурой остаются активными до своего естественного истечения.

### 8.3. Связь с конкретными объявлениями
В первой версии не реализуется связь между конкретным объявлением на торговой площадке и индексом цены этой же номенклатуры. На странице объявления отсылка к индексу не отображается. На странице индекса не отображается перечень активных объявлений по этой номенклатуре.

Подключение взаимной отсылки — открытый вопрос для последующих версий.
