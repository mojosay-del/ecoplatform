## 1. Назначение раздела
Настоящий документ фиксирует устройство раздела «Индексы цен» административной панели — рабочего пространства контент-менеджера и администратора, в котором создаются и редактируются индексы, ведётся история значений, управляется справочник категорий номенклатуры и настраиваются пороги тренда.

Документ продолжает [overview.md](./overview.md), раздел 9, и развивает workflow до уровня экранов и операций.

Документ определяет:
- размещение раздела в админ-панели;
- список индексов и его фильтры;
- редактор индекса с историей значений;
- управление категориями номенклатуры (только администратор);
- настройку порогов тренда (только администратор);
- workflow перехода между состояниями;
- удаление черновиков и удаление опубликованного;
- журналирование действий.

## 2. Размещение в админ-панели
Раздел «Индексы цен» — пункт бокового меню административной панели (`07-platform/admin-panel.md`, раздел 6). Доступен платформенным сотрудникам с ролями `content_manager` и `admin`.

При нажатии на пункт меню открывается экран списка индексов (раздел 3). Внутри раздела доступны переходы:
- к редактору индекса (раздел 5);
- к экрану управления категориями номенклатуры (раздел 4) — только администратору;
- к экрану настроек порогов тренда (раздел 8) — только администратору.

Управление справочником номенклатуры (добавление, переименование, деактивация конкретных видов сырья) выполняется через отдельный раздел админ-панели (`07-platform/admin-panel.md`, раздел 13.2). Из раздела «Индексы цен» доступна быстрая ссылка на справочник номенклатуры для удобства.

## 3. Список индексов

### 3.1. Состав экрана
Экран отображает все индексы платформы независимо от их состояния (`draft` и `published`). Каждая строка списка содержит:
- название номенклатуры и её код;
- категория номенклатуры (вкладка);
- состояние публикации индекса;
- число значений в истории;
- дата последнего значения с `date ≤ today`;
- момент последней редакции метаданных индекса;
- автор последней редакции (контент-менеджер или администратор);
- ссылка «Редактировать» — переход к редактору индекса (раздел 5).

### 3.2. Фильтры
- по состоянию публикации (`draft`, `published`, оба);
- по категории номенклатуры (одна или несколько);
- свободный поиск по названию или коду номенклатуры;
- фильтр «индексы без значений за последние N дней» — для контроля актуальности данных. По умолчанию N = 30; параметр конфигурации.

### 3.3. Сортировка
По умолчанию — по моменту последней редакции, новые сверху. Альтернативные сортировки — по названию номенклатуры, по дате последнего значения, по числу значений.

### 3.4. Действие «Создать индекс»
Кнопка «Создать индекс» открывает диалог выбора номенклатуры:
- поле автодополнения с поиском по коду и названию;
- список номенклатур с активным признаком `isActive = true`, не имеющих индекса (или имеющих индекс в состоянии `removed` — открытый вопрос для последующих версий, в первой версии не предусматривается);
- кнопка «Создать индекс» применяется к выбранной номенклатуре.

После выбора создаётся пустой `PriceIndex` в состоянии `draft` для выбранной номенклатуры; пользователь автоматически переходит в редактор индекса (раздел 5).

Создать индекс по номенклатуре, у которой уже есть индекс, нельзя. Платформа отображает существующий индекс в выпадающем списке как «Уже есть» и блокирует выбор.

## 4. Управление категориями номенклатуры

### 4.1. Кому доступен экран
Экран «Управление категориями» доступен только администратору платформы. Контент-менеджер не имеет к нему доступа.

Контент-менеджер видит категории как существующий справочный список при работе с индексами и не может его менять.

### 4.2. Состав экрана
Экран отображает справочник категорий в виде упорядоченной таблицы:
- название категории (`name`);
- слаг (`slug`);
- порядок (`position`);
- признак активности (`isActive`);
- число номенклатур в категории;
- число опубликованных индексов в категории;
- момент создания;
- автор создания;
- кнопки операций: «Переименовать», «Деактивировать»/«Активировать», «Удалить».

Над таблицей — кнопка «Добавить категорию» и поле поиска по названию.

### 4.3. Добавление категории
Кнопка «Добавить категорию» открывает диалог:
- поле «Название» (`name`);
- поле «Порядок» (`position`) — по умолчанию равен последнему `position` плюс один; администратор может задать другое значение, при этом порядки соседних категорий пересчитываются автоматически.

После сохранения создаётся запись `NomenclatureCategory` со статусом `isActive = true`. Действие фиксируется в `AdminActionLog`.

### 4.4. Переименование категории
Кнопка «Переименовать» открывает диалог с полем нового названия. После подтверждения:
1. Платформа проверяет, что новое название не дублирует другую существующую категорию (по нормализованной форме);
2. Меняет поле `name`;
3. Пересчитывает `slug`;
4. Сохраняет старый слаг как редирект на новый (на уровне маршрутизации платформы).

Действие фиксируется в `AdminActionLog`.

### 4.5. Управление порядком вкладок
Порядок категорий задаётся перетаскиванием строк в таблице. При перетаскивании пересчитываются поля `position` затронутых категорий. Изменения сохраняются автоматически.

### 4.6. Деактивация и активация
Кнопка «Деактивировать» переводит категорию в `isActive = false`. Деактивированная категория не отображается как вкладка в публичной части индексов и недоступна для выбора при создании или редактировании номенклатуры. Существующие связи с номенклатурами сохраняются.

Активация — обратная операция. Действия фиксируются в `AdminActionLog`.

### 4.7. Удаление категории
Кнопка «Удалить» открывает диалог подтверждения. Удаление возможно только для категории, в которой нет активных номенклатур.

Если в категории есть активные номенклатуры, удаление блокируется с сообщением «В категории {name} есть {N} активных номенклатур. Переместите их в другую категорию или деактивируйте перед удалением».

После удаления категории:
- сами номенклатуры остаются (с пустым `categoryId`, требующим переназначения);
- связь с индексами сохраняется (но индексы становятся невидимыми в публичной части до переназначения категории номенклатур).

Действие фиксируется в `AdminActionLog`.

## 5. Редактор индекса

### 5.1. Состав экрана
Экран редактора индекса разделён на две колонки:
- **Левая колонка** — параметры индекса:
  - название номенклатуры (нередактируемо; для смены номенклатуры индекс пересоздаётся);
  - код номенклатуры (нередактируемо);
  - категория (отображается как информационное поле; меняется через карточку номенклатуры в `07-platform/admin-panel.md`, раздел 13.2);
  - единица измерения (отображается как информационное поле; меняется через карточку номенклатуры);
  - описание индекса (`description`) — служебное поле для контент-менеджера, не отображается в публичной части;
  - состояние публикации с кнопкой управления (раздел 7).
- **Правая колонка** — таблица значений (раздел 6) и форма добавления нового значения.

Под колонками — кнопка «Просмотреть как пользователь» (раздел 5.2).

### 5.2. Предпросмотр
Кнопка «Просмотреть как пользователь» открывает превью карточки индекса в виде, идентичном публичному интерфейсу (см. [public-ui.md](./public-ui.md), раздел 4). Превью доступно для индексов в любом состоянии (включая `draft`); открывается в отдельной модалке с пометкой «Режим предпросмотра».

## 6. Управление значениями

### 6.1. Таблица значений
В правой колонке редактора отображается таблица значений:
- столбец «Дата» — отображается в формате «дд.мм.гггг»;
- столбец «Цена» — отображается с разделителем тысяч и единицей измерения (например, «14 250 ₽/т»);
- столбец «Изменено» — момент последней редакции значения;
- столбец «Кто» — автор последней редакции (контент-менеджер или администратор);
- кнопки операций на строке: «Редактировать», «Удалить».

Сортировка по умолчанию — по дате убыванию (последние сверху). Возможна сортировка по дате возрастанию.

Над таблицей — фильтр по диапазону дат и поле поиска (по дате).

### 6.2. Добавление значения
Над таблицей — форма быстрого добавления значения:
- поле даты — по умолчанию текущая дата, с возможностью выбрать любую;
- поле цены — числовой ввод с автоматическим форматированием;
- кнопка «Добавить».

При нажатии «Добавить»:
1. Платформа проверяет валидность данных: корректная дата, цена > 0;
2. Если на указанную дату уже существует значение, отображается диалог «Заменить существующее значение от {дата} ({цена})?» с кнопками «Заменить» и «Отмена»;
3. При подтверждении замены значение обновляется, `lastUpdatedAt` устанавливается в текущее время;
4. При подтверждении нового значения создаётся запись `PriceIndexValue`;
5. Таблица значений обновляется немедленно.

### 6.3. Редактирование значения
Кнопка «Редактировать» на строке открывает форму с полями даты и цены, заполненными текущими значениями. После сохранения:
- если дата не изменилась — обновляется только цена;
- если дата изменилась и на новую дату нет другого значения — обновляются оба поля;
- если на новую дату уже существует другое значение — отображается диалог «Заменить существующее значение от {новая дата}?»; при подтверждении старая запись (с прежней датой) удаляется, на новой дате обновляется существующее значение.

### 6.4. Удаление значения
Кнопка «Удалить» на строке открывает диалог подтверждения «Удалить значение от {дата} ({цена})?». При подтверждении запись `PriceIndexValue` удаляется.

Удаление значения не блокируется состоянием публикации индекса (как `draft`, так и `published`). Контент-менеджер может удалять как ошибочно введённые значения, так и устаревшие/неактуальные.

### 6.5. Массовый импорт значений
Импорт значений из CSV или иной таблицы в первой версии не реализуется. Контент-менеджер вводит значения вручную через форму добавления (раздел 6.2).

Реализация массового импорта — открытый вопрос для последующих версий.

## 7. Workflow публикации

### 7.1. Состояния и переходы
Состояния индекса: `draft`, `published` (см. [indices.md](./indices.md), раздел 3.3).

Переходы:
- `draft → published` — выполняется контент-менеджером или администратором;
- `published → draft` — выполняется только администратором.

### 7.2. Кнопка «Опубликовать»
В левой колонке редактора рядом с состоянием публикации — кнопка «Опубликовать» (если индекс в `draft`).

При нажатии:
1. Платформа выполняет валидацию (см. [indices.md](./indices.md), раздел 6.1);
2. При успешной валидации индекс переводится в `published`, в поле `firstPublishedAt` подставляется текущее время (если поле пустое);
3. При ошибках валидации отображается список ошибок с пояснениями: «У индекса нет ни одного значения с датой ≤ сегодня», «Номенклатура деактивирована», «Значение от {дата} имеет недопустимую цену» и т. п.

### 7.3. Кнопка «Снять с публикации»
Кнопка «Снять с публикации» отображается только администратору. При нажатии открывается диалог с обязательным полем «Причина снятия с публикации». После подтверждения:
- индекс переводится в `draft`;
- индекс исчезает из публичной части;
- действие фиксируется в `AdminActionLog`.

### 7.4. Двухстадийный workflow
Создание и редактирование выполняются непосредственно контент-менеджером без отдельной стадии согласования. Промежуточное состояние «на ревью» в первой версии не предусматривается.

## 8. Настройки порогов тренда

### 8.1. Кому доступен экран
Экран настроек порогов тренда доступен только администратору. Размещён в служебных настройках раздела (`07-platform/admin-panel.md`, раздел 18).

### 8.2. Параметры
- `stagnationThreshold` — порог стагнации в процентах. Значение по умолчанию — `1.0`. Допустимый диапазон — `0.1` — `10.0`.

При значении `stagnationThreshold = 1.0`:
- `weeklyChange > +1.0%` → плашка «Рост»;
- `−1.0% ≤ weeklyChange ≤ +1.0%` → плашка «Стагнация»;
- `weeklyChange < −1.0%` → плашка «Падение».

### 8.3. Применение
Изменение порога применяется глобально ко всем индексам. Эффект отображается на следующих обращениях пользователей к публичной части.

Изменения фиксируются в `AdminActionLog` с указанием прежнего и нового значения.

## 9. Удаление индексов

### 9.1. Удаление черновиков
Контент-менеджер удаляет:
- индекс в состоянии `draft`, не имевший ранее опубликованных редакций (`firstPublishedAt` пуст).

Удаление подтверждается простым диалогом. Каскадно удаляются все значения этого индекса. Действие в `AdminActionLog` не записывается.

### 9.2. Удаление опубликованных индексов
Удаление опубликованных индексов выполняется только администратором. При удалении:
- открывается диалог с обязательным полем «Причина удаления»;
- удаление выполняется каскадно по правилам [indices.md](./indices.md), раздел 8.1;
- действие фиксируется в `AdminActionLog`.

После удаления индекса соответствующая номенклатура остаётся в справочнике без индекса. На вкладке группы в публичной части номенклатура не отображается (поскольку отображаются только номенклатуры с опубликованными индексами).

## 10. Журналирование

### 10.1. Что фиксируется в `AdminActionLog`
Следующие действия в разделе «Индексы цен» админ-панели фиксируются в `AdminActionLog` (`07-platform/admin-panel.md`, раздел 16):
- публикация индекса (`draft → published`);
- снятие индекса с публикации (`published → draft`);
- удаление опубликованного индекса администратором;
- создание новой категории номенклатуры (только администратором);
- переименование категории номенклатуры;
- деактивация и активация категории;
- удаление категории;
- изменение порядка категорий;
- изменение порогов тренда (раздел 8.3).

### 10.2. Что не фиксируется
- создание индексов как черновиков;
- редактирование служебного описания индекса;
- добавление, редактирование, удаление отдельных значений `PriceIndexValue`;
- удаление контент-менеджером собственных черновиков индексов;
- быстрая смена сортировки и фильтров в списке индексов.

Эти операции — массовые контентные правки. Они журналируются на уровне общего журнала редактирования контентных сущностей, отдельного от `AdminActionLog`. Конкретный журнал контентных правок и его правила хранения — открытый вопрос для последующих версий.

## 11. Открытые вопросы первой версии

### 11.1. Конкурентное редактирование
Возможность одного контент-менеджера временно занять индекс в редактировании (lock) и предотвратить одновременное редактирование двумя сотрудниками в первой версии не реализуется. При одновременном редактировании двух пользователей побеждает тот, чьё сохранение прошло позже.

### 11.2. Массовый импорт значений
Импорт значений из CSV/Excel в первой версии не реализуется (раздел 6.5).

### 11.3. История изменений значения
Хранение истории редакций значения с возможностью отката (например, «значение от 15 июня было 14 250 ₽/т, изменено на 14 280 ₽/т 16 июня») в первой версии не реализуется. Изменение перезаписывает предыдущее.

### 11.4. Уведомления контент-менеджеру о устаревших данных
Автоматическое напоминание контент-менеджеру об индексах без обновлений за N дней (например, «Индекс «Картон» не обновлялся 30 дней») в первой версии не реализуется. Контент-менеджер использует фильтр в списке индексов (раздел 3.2) для контроля актуальности.
