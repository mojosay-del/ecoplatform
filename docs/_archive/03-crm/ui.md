## 1. Интерфейс CRM

### 1.1. Главный экран кабинета заготовителя
При входе пользователь видит сводный главный экран с блоками, доступными его роли:
- сводка по складу (остатки по видам сырья, готовность к отгрузке)
- мои задачи на сегодня и просроченные
- активные сделки закупки по ключевым этапам
- активные сделки продажи по ключевым этапам
- последние поступления сырья
- текущая дебиторка (только для собственника и бухгалтера)

Главный экран даёт мгновенный ответ на вопрос «что сейчас происходит в компании».

### 1.2. Экран контрагентов
Экран контрагентов — единый список с переключателями «Поставщики», «Покупатели», «Все», «Архив».

Поля в списке: наименование, тип, телефон, ответственный, последняя сделка, общий объём.

Финансовые поля, включая общую сумму и текущую дебиторку, доступны только собственнику и бухгалтеру.

Фильтры: по категории поставщика, типу покупателя, ответственному, периоду последнего взаимодействия, виду сырья.

Сортировка: по алфавиту, по дате последнего взаимодействия, по объёму. Сортировка по дебиторке доступна только собственнику и бухгалтеру.

### 1.3. Экраны сделок
Сделки закупки и сделки продажи представлены в двух режимах:
- канбан-доска по этапам воронки
- список с фильтрами

Канбан используется как основной рабочий интерфейс. Список — как альтернативный для углублённого анализа.

На канбан-доске каждая карточка сделки содержит: контрагента, дату ближайшего действия, ответственного, ключевые параметры (номенклатура, вес, сумма), индикатор задач и сроков.

### 1.4. Экран поступления сырья
Экран поступления сырья оптимизирован для работы с мобильного устройства или планшета у весов. На нём видна форма поступления сырья и список поступлений за сегодня.

Ключевое требование — минимальное число касаний для оформления одного поступления в типовом сценарии.

### 1.5. Экран склада
Экран склада отображает остатки по видам сырья в двух состояниях. Описан в разделе 11.3 (см. [warehouse.md](./warehouse.md)).

По каждому виду сырья доступны кнопки быстрых действий: Пересортировать, Спрессовать, Списать.

### 1.6. Экран задач
Экран задач — «Мои задачи» для рядового пользователя и «Все задачи» для собственника и руководителя. Описан в разделе 13.5 и 13.6 (см. [tasks-and-comms.md](./tasks-and-comms.md)).

### 1.7. Экран дебиторки
Экран дебиторки описан в разделе 14.4 (см. [finance.md](./finance.md)) и доступен только собственнику и бухгалтеру.

### 1.8. Экран документов
Экран документов — единый список с фильтрами. Описан в разделе 15.4 (см. [documents.md](./documents.md)).

### 1.9. Экран отчётов
Экран отчётов доступен только собственнику и бухгалтеру.

Экран отчётов в первой версии содержит:
- отчёт «Доходы минус расходы» по периодам и разрезам
- отчёт по обороту сырья
- отчёт по поставщикам
- отчёт по покупателям
- отчёт по дебиторке

Отчёты представлены в виде таблиц с возможностью экспорта в Excel.

### 1.10. Настройки кабинета
Экран настроек CRM заготовителя содержит:
- общий прайс закупки по видам сырья
- общий прайс продажи по видам сырья
- настройки автозадач по этапам воронок
- пороги «готово к отгрузке» по видам сырья
- настройки ролей и сотрудников
- хранение шаблонов документов как файлов без автогенерации и без автоподстановки данных (правила работы с шаблонами — см. раздел 1.11)

### 1.11. Шаблоны документов CRM

В настройках CRM компания-заготовитель может хранить набор шаблонов документов (договор, ТТН, УПД, спецификация, заявка на перевозку и иные) как файлов для последующего использования при оформлении сделок. Платформа в первой версии не выполняет автогенерацию документов и не подставляет в шаблоны данные сделки автоматически.

Полные правила работы с шаблонами — `07-platform/storage.md`, раздел 5.8 (шаблоны документов CRM). В частности:

- уровень доступа — `company-private` (доступны только сотрудникам компании-заготовителя в соответствии с ролевой моделью CRM);
- максимальное число шаблонов на компанию — 50;
- максимальный размер одного шаблона — 25 МБ;
- допустимые форматы — PDF, DOCX, XLSX, ODT, ODS;
- все загружаемые файлы проходят обязательную антивирусную проверку (см. `07-platform/storage.md`, раздел 7);
- при удалении компании шаблоны удаляются.

Загружать, заменять и удалять шаблоны могут роли с доступом к настройкам CRM (собственник, руководитель, бухгалтер) — в соответствии с ролевой моделью CRM (`03-crm/roles.md`).
