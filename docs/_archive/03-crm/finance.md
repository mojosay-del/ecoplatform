## 1. Финансовый контур CRM

### 1.1. Границы финансового контура в первой версии
Финансовый контур в первой версии ограничен задачами фиксации оплат и контроля дебиторки. Он не заменяет бухгалтерскую систему и не формирует бухгалтерские проводки.

Основные задачи:
- фиксация оплат по каждому поступлению сырья (закупка)
- фиксация оплат по каждой сделке продажи
- контроль дебиторки по покупателям (переработчикам и трейдерам)
- простейший отчёт «доходы минус расходы» за период

### 1.2. Оплата по закупке
Каждое поступление сырья создаёт связанную запись оплаты с атрибутами:
- сумма
- форма оплаты (наличные, карта физического лица, безналичный расчёт)
- статус (оплачено, ожидает оплаты, частично оплачено, отменено)
- дата оплаты
- ответственный за оплату (собственник или бухгалтер)
- номер платёжного документа или транзакции (опционально)
- комментарий

Оплата по закупке привязана 1:1 к записи поступления сырья. Сделка закупки агрегирует свои поступления и, соответственно, их оплаты.

При наличных и карточных оплатах статус «оплачено» устанавливается сразу в момент поступления сырья. При безналичных — статус «ожидает оплаты», фактическая оплата фиксируется собственником или бухгалтером позже.

### 1.3. Оплата по продаже
Каждая сделка продажи может иметь одну или несколько связанных записей оплаты с атрибутами:
- сумма
- способ (обычно безналичный расчёт)
- статус (ожидается, поступило, частично поступило, просрочено)
- ожидаемая дата оплаты (рассчитывается от даты приёмки на стороне покупателя плюс срок отсрочки)
- фактическая дата оплаты
- комментарий

Оплата фиксируется вручную собственником или бухгалтером.

### 1.4. Дебиторка
Дебиторка — это сумма, которую покупатели должны заготовителю по проведённым отгрузкам.

Экран дебиторки отображает:
- список покупателей с ненулевой задолженностью
- по каждому покупателю: сумма долга, срок просрочки, дата последней отгрузки, дата ожидаемой оплаты, контактное лицо, телефон, ответственный менеджер
- общая сумма дебиторки
- разбивка дебиторки по срокам (текущая, просроченная до 7 дней, 7–30 дней, более 30 дней)

Дебиторка строится как по переработчикам, так и по трейдерам, без различия в логике.

По каждому покупателю можно развернуть список конкретных сделок, формирующих дебиторку.

Экран дебиторки должен предоставлять действия: «Позвонить», «Создать задачу напоминания», «Написать покупателю».

### 1.5. Отчёт доходы минус расходы
Отчёт «Доходы минус расходы» за выбранный период — упрощённый денежный отчёт:
- доходы: сумма фактически поступивших оплат от покупателей за период
- расходы: сумма фактически произведённых оплат поставщикам за период
- итог: доходы минус расходы

Отчёт отражает денежный поток за период и предназначен для быстрой самодиагностики заготовителя: «в плюсе или в минусе за период». Отчёт не учитывает остатки сырья на складе, себестоимость отгружаемой партии, операционные расходы компании (аренда, зарплата, коммунальные, логистика, кредиты) и иные бухгалтерские аспекты.

В первой версии отчёт доступен в разрезах:
- за период (день, неделя, месяц, произвольный)
- по виду сырья
- по покупателю
- по поставщику

Для полноценного финансового учёта заготовитель использует внешние инструменты.

### 1.6. Рабочее место бухгалтера
Для роли «бухгалтер» в кабинете заготовителя предусматривается упрощённое рабочее место со следующим составом:
- список поступлений с фильтром «ожидает оплаты» (безналичные поступления, которые надо оплатить поставщикам)
- список сделок продажи с фильтром «ожидает оплаты» (поступления, которые надо проверить по расчётному счёту)
- экран дебиторки
- экран документов с фильтрами по контрагентам, сделкам и типам
- отчёты (доходы минус расходы, сверка по контрагентам)

Список поступлений с фильтром «ожидает оплаты» фактически отображает текущие обязательства заготовителя перед поставщиками (кредиторку) и позволяет контролировать просрочки платежей поставщикам. Экран доступен бухгалтеру и собственнику.

Бухгалтер имеет доступ к финансовому контуру, остаткам и корректировкам остатков. Бухгалтер не имеет управленческих прав.

### 6.7. Минимальный финансовый контур
Финансовый контур первой версии ограничен:
- фиксацией оплат по поступлениям сырья
- фиксацией оплат от покупателей
- дебиторской задолженностью по покупателям
- простым отчётом «доходы минус расходы»

Интеграции с банками, 1С, онлайн-кассами и автогенерация первичных документов не входят в первую версию.
