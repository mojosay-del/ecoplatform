## 1. Контрагенты

### 1.1. Общее определение
Контрагент — это любое лицо или организация, с которой заготовитель взаимодействует в рамках коммерческой деятельности через CRM.

Контрагент может являться:
- поставщиком (источник сырья для заготовителя)
- покупателем (переработчик или трейдер, которому заготовитель продаёт сырьё)
- одновременно поставщиком и покупателем
- прочим контрагентом, если такой тип явно зафиксирован в утверждённой спецификации

В первой версии CRM оперирует только двумя функциональными типами контрагентов: поставщик и покупатель. Прочие типы не входят в первую версию.

### 1.2. Юридические формы контрагентов
Контрагент может быть:
- физическим лицом
- индивидуальным предпринимателем
- обществом с ограниченной ответственностью
- иным юридическим лицом

Для каждой формы должен быть предусмотрен соответствующий набор реквизитов.

Для физического лица CRM должна хранить как минимум: имя и телефон. Дополнительно на усмотрение заготовителя в карточке могут храниться фамилия, отчество, адрес проживания, паспортные данные и иные реквизиты. Эти поля необязательные.

Для ИП и юридических лиц CRM должна хранить как минимум: наименование, ИНН, КПП для юридических лиц, ОГРН или ОГРНИП, адрес, расчётный счёт, банковские реквизиты, контактное лицо, телефон, электронную почту.

### 1.3. Общая карточка контрагента
В карточке контрагента, независимо от его типа, должны присутствовать:
- наименование или имя
- тип контрагента (поставщик, покупатель, одновременно)
- юридическая форма
- реквизиты в зависимости от формы
- контактные данные
- адрес или адреса
- ответственный менеджер внутри компании-заготовителя
- дата создания карточки
- дата последнего взаимодействия
- рабочие заметки в свободной форме
- закреплённые задачи
- история взаимодействий
- список сделок
- список оплат
- прикреплённые документы
- рейтинг и отзывы, подтягиваемые из платформенных модулей, если они доступны по данному контрагенту

Заготовитель видит только те данные контрагента, которые внесены им самим или его сотрудниками.

### 1.4. Карточка поставщика
Карточка поставщика расширяется специфическими полями закупочного контура:
- категория поставщика (магазин, аптека, школа, рынок, предприятие, физическое лицо, иной заготовитель, иная организация)
- обычное сырьё, которое сдаёт поставщик, с возможностью указания нескольких позиций
- типовой объём единичной сдачи по каждому виду сырья
- индивидуальная цена закупки по каждому виду сырья, если она отличается от общего прайса
- способ оплаты по умолчанию (наличные, карта физического лица, безналичный расчёт)
- условия вывоза (самовывоз поставщика, выездной забор силами заготовителя, смешанный)
- признак постоянного клиента
- признак платного вывоза (клиент платит за вывоз, условие бесплатного вывоза, закупка за деньги)
- адреса точек забора, если их несколько
- контактные лица на каждой точке
- комментарии менеджера по особенностям работы
- автоматически вычисляемая статистика: общий объём закупок по каждому виду сырья, средняя цена закупки, последняя дата вывоза

Карточка поставщика должна быть максимально лёгкой для заполнения. Обязательные поля минимизируются до наименования, телефона и способа связи. Всё остальное заполняется по мере накопления истории.

### 1.5. Карточка покупателя
Карточка покупателя расширяется специфическими полями продажного контура:
- тип покупателя (переработчик, трейдер, иной)
- адрес приёмки (место, куда везётся сырьё)
- виды сырья, которые принимает покупатель, с возможностью указания нескольких позиций
- текущая закупочная цена по каждому виду сырья
- условия приёмки по каждому виду сырья (требования к качеству, максимальная влажность, допустимый засор, минимальный объём партии)
- типовая отсрочка оплаты
- способ оплаты
- условие цены работы с покупателем: «цена с места» (покупатель забирает сырьё с площадки заготовителя) или «цена на воротах» (заготовитель везёт сырьё к покупателю)
- режим работы приёмки
- контактное лицо на приёмке
- комментарии менеджера по особенностям работы, в том числе по приёмщику, качеству приёмки, частоте срезов веса
- автоматически вычисляемая статистика: общий объём продаж, средняя цена, средняя фактическая цена после приёмки с учётом срезов, средняя задержка оплаты, количество отгрузок за период, текущая дебиторка

Карточка покупателя также должна быть лёгкой для заполнения в минимальном варианте. Остальные поля заполняются по мере использования.

### 1.6. Создание карточки контрагента
Карточка контрагента может быть создана следующими способами:
- вручную через отдельную форму «Новый контрагент»
- автоматически при создании сделки, если выбранного контрагента ещё нет в базе

При автоматическом создании из сделки система просит заполнить только минимальный набор полей. Остальные поля заполняются пользователем при наличии данных.

### 1.7. Редактирование и архивация
Карточка контрагента может редактироваться ответственным менеджером, руководителем или собственником компании в соответствии с ролевой моделью.

Карточка контрагента не удаляется физически. Вместо удаления используется архивация с сохранением всей истории сделок, оплат, задач и документов. Архивированная карточка не отображается в основных списках, но доступна в архиве.

Повторное восстановление карточки из архива допускается.

### 1.8. Дубликаты
CRM должна предупреждать о возможных дубликатах контрагента при создании новой карточки. Критерии определения дубликата: совпадение ИНН, совпадение телефона, совпадение наименования при совпадении адреса.

Автоматическое объединение дубликатов в первой версии не предусматривается. Решение принимается пользователем.
