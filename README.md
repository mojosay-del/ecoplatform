# ЭкоПлатформа MVP

Этот репозиторий теперь содержит не только продуктовую документацию, но и стартовую кодовую базу MVP.

## Карта проекта

- `apps/web` — пользовательский и административный интерфейс на Next.js. Здесь находятся публичные разделы `/news`, `/indices`, `/education`, `/knowledge-base`, личный кабинет `/account` и админка `/admin`.
- `apps/api` — backend на NestJS. Здесь находятся авторизация, demo-доступ, ручная подписка, CMS, контентные API и Prisma-схема.
- `packages/shared` — общие типы, статусы, правила доступа, DTO-схемы и небольшие функции, которые одинаково нужны web и API.
- `docs` — утверждённая и черновая продуктовая документация.
- `PROJECT_STATUS.md` — текущий ход разработки: что сделано, что проверено, где остановились и что делать дальше.

## Локальный запуск

1. Установить зависимости:

```bash
pnpm install
```

2. Скопировать переменные окружения:

```bash
cp .env.example .env
```

3. Запустить PostgreSQL локально или через будущий `docker-compose.yml`.

4. Подготовить базу:

```bash
pnpm --filter @ecoplatform/api prisma:generate
pnpm --filter @ecoplatform/api prisma:migrate
pnpm --filter @ecoplatform/api seed
```

5. Запустить разработку:

```bash
pnpm dev
```

## Как читать код без опыта разработки

В сложных файлах есть русские пояснения: что делает файл, почему есть конкретная бизнес-проверка и какие сценарии она обслуживает. Комментарии не дублируют каждую строку, а объясняют важные решения: demo-доступ, роли, публикацию контента, расчёт индексов и структуру блоков.
