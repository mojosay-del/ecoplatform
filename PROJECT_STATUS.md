# Ход разработки ЭкоПлатформы MVP

## Текущий этап

Этап: старт кодовой базы после согласования MVP.

Цель текущего этапа: поднять Turborepo, общий shared-пакет, NestJS API, Next.js web, базовую Prisma-схему и первые MVP-экраны.

## Что уже сделано

- Создан корневой Turborepo-каркас.
- Добавлены рабочие области `apps/web`, `apps/api`, `packages/shared`.
- Добавлен этот файл статуса, чтобы следующий разработчик или другой ИИ видел, где остановилась работа.
- Добавлен русский `README.md` с картой проекта.

## Архитектурные решения

- Monorepo: Turborepo + pnpm workspaces.
- Frontend: Next.js App Router.
- Backend: NestJS.
- База данных: PostgreSQL через Prisma.
- Общие типы и правила доступа: `packages/shared`.
- На первом dev-этапе платёжный шлюз не подключается: компания создаётся в `demo`, а администратор вручную переводит её в `active`.
- Demo-доступ приравнен к базовой подписке, но ограничен 24 часами и уникальной парой email + телефон.

```bash

```
